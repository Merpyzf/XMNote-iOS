# 时间线 iOS ↔ Android 对齐情况

评估日期：2026-03-10

## 功能点清单

1. 时间线入口页可用性
- 功能点：在主流程中可进入并使用时间线页面。
- iOS 行为：`ReadingContainerView` 已在 `timeline` 子页挂载 `ReadingTimelineView`，进入后直接展示正式时间线页面。`ReadingTimelineView` 会在 `.task` 中创建 `TimelineViewModel` 并触发首次加载。
- Android 行为：`StatisticsContainerFragment` 通过 `ViewPager` 挂载 `TimelineFragment`，进入后直接展示完整时间线页面。
- 对齐状态：已对齐。
- 双端代码证据：

2. 多源聚合查询与按时间排序 / 按日分组
- 功能点：跨表聚合书摘、计时、状态、相关内容、书评、打卡并按时间倒序输出，最终按自然日分组。
- iOS 行为：`TimelineRepository` 已按分类路由到 6 路查询，Swift 侧统一合并排序并返回 `[TimelineSection]`。`TimelineViewModel` 持有 `sections` 作为页面数据源。
- Android 行为：`TimelineRepository.getTimelineDataList` 在事务中聚合多路数据后排序，并生成分组信息。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/Data/Repositories/TimelineRepository.swift:21`、`xmnote/Data/Repositories/TimelineRepository.swift:58`、`xmnote/ViewModels/Reading/TimelineViewModel.swift:18`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/data/src/main/java/com/merpyzf/data/repository/TimelineRepository.kt:71`、`/Users/wangke/Workspace/AndroidProjects/XMNote/data/src/main/java/com/merpyzf/data/repository/TimelineRepository.kt:185`

3. 分类过滤能力（全部 / 书摘 / 状态 / 相关 / 书评 / 计时 / 打卡）
- 功能点：切换分类后重新拉取列表与日历标记。
- iOS 行为：`TimelineEventCategory` 定义了 7 类筛选 + 全部；`TimelineViewModel.selectCategory(_:)` 会清空标记缓存、重查列表并预加载月份标记；页面顶部分类入口由时间线正式页面承载。
- Android 行为：`TimelineFragment` 提供分类菜单，切换后调用 `changeTimelineShowDataCategory` 并刷新列表。
- 对齐状态：已对齐。
- 双端代码证据：

4. 日历选日与“返回今天”联动刷新
- 功能点：选中日期后刷新列表，并支持一键回到今天。
- iOS 行为：`TimelineCalendarPanel` 在 `onDaySelection` 中提交新的 `selectedDate`；顶部“今”按钮会跳转并回到当天；`TimelineViewModel.selectDate(_:)` 负责实际重查。
- Android 行为：`OnCalendarSelectListener` 与“回到今天”按钮都会触发 `presenter.getTimelineData(...)`。
- 对齐状态：已对齐。
- 双端代码证据：

5. 月份跳转与日历标记聚合

- Android 行为：时间线页面在月份切换时调用 `presenter.getCalendarActiveData(...)`，更新对应月份的活跃信息。

6. 数据范围策略（当天 / 过去一个月 / 半年 / 一年 / 全部）
- 功能点：当选中“今天”时按时间范围配置决定查询窗口。
- iOS 行为：`TimelineViewModel` 已通过 `@AppStorage("timelineDataShowRange")` 读取范围设置，并在今天场景下按 `0/1/2/3/4` 对应 `当天/31天/183天/366天/全部` 计算查询范围；当前默认值是 `2`。但页面上尚无显式设置入口。
- Android 行为：`SpSettingHelper.getTimeLineDataShowRange()` 提供同一组配置，并参与 `TimelineRepository.getTimelineDataList` 的范围计算。
- 对齐状态：未对齐。
- 双端代码证据：

## 总结
- 本轮之后，iOS 时间线已经从 placeholder 阶段进入正式可用阶段，核心浏览能力与 Android 已基本对齐。
- 当前主要差距集中在“设置入口”和“二级操作链路”，属于后续功能补齐，而不是当前设计倒退。
