# 时间线 iOS ↔ Android 对齐情况

评估日期：2026-03-08

## 功能点清单

1. 时间线入口页可用性
- 功能点：在主流程中可进入并使用时间线页面。
- iOS 行为：在读容器的 `timeline` 子页当前仍渲染 `TimelinePlaceholderView` 空态占位。
- Android 行为：统计容器直接挂载 `TimelineFragment`，进入后展示完整时间线列表与交互。
- 对齐状态：未对齐。
- 双端代码证据：
  - iOS：`xmnote/Views/Reading/ReadingContainerView.swift:54`、`xmnote/Views/Reading/TimelinePlaceholderView.swift:18`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/main/container/StatisticsContainerFragment.kt:79`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/main/fragment/statistics/TimelineFragment.kt:108`
- 评估结论：设计倒退。
- Android 反向建议：
  - 是否吸收：不适用（应优先补齐 iOS 主流程能力）。
  - 建议优先级：高。
  - 风险：高（当前 iOS 用户无法使用时间线核心能力）。

- Android 反向建议：
  - 是否吸收：不适用（应在 iOS 侧补齐聚合查询能力）。
  - 建议优先级：高。
  - 风险：高（时间线数据主价值点缺失）。
