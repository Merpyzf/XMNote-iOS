# 时间线功能设计文档

## 实施阶段

### Phase 1: 领域模型定义

**目标：** 定义时间线所有类型的数据结构和 Repository 协议。

#### 新建 `xmnote/Domain/Models/TimelineModels.swift`

统一事件模型，用 enum associated values 建模 7 种事件：

```swift
enum TimelineEventKind {
    case note(TimelineNoteEvent)
    case readTiming(TimelineReadTimingEvent)
    case readStatus(TimelineReadStatusEvent)
    case checkIn(TimelineCheckInEvent)
    case review(TimelineReviewEvent)
    case relevant(TimelineRelevantEvent)
    case relevantBook(TimelineRelevantBookEvent)
}

struct TimelineEvent: Identifiable {
    let id: String               // "{kind}_{recordId}" 全局唯一
    let kind: TimelineEventKind
    let timestamp: Int64         // 毫秒时间戳（排序键）
    let bookId: Int64
    let bookName: String
    let bookAuthor: String
    let bookCover: String
}

struct TimelineSection: Identifiable {
    let id: String               // "yyyy-MM-dd"
    let date: Date
    let events: [TimelineEvent]
}

enum TimelineEventCategory: CaseIterable {
    case all, note, readStatus, relevant, review, readTiming, checkIn
}
```

各 Event 子结构携带类型特有的字段（content/idea/elapsedSeconds/statusId/amount 等）。

#### 修改 `xmnote/Domain/Repositories/RepositoryProtocols.swift`

新增 `TimelineRepositoryProtocol`:

```swift
protocol TimelineRepositoryProtocol {
    func fetchTimelineEvents(
        start: Int64, end: Int64,
        category: TimelineEventCategory
    ) async throws -> [TimelineSection]
}
```

**可复用资源：** 无需新增 Record，全部 6 张表的 Record 已就绪（NoteRecord / ReadTimeRecordRecord / BookReadStatusRecordRecord / CheckInRecordRecord / ReviewRecord / CategoryContentRecord）。

### Phase 2: TimelineRepository 实现

**目标：** GRDB 多表聚合查询，合并排序，按日分组。

#### 新建 `xmnote/Data/Repositories/TimelineRepository.swift`

核心逻辑（对齐 Android `TimelineRepository.getTimelineDataList`）：

1. **书摘查询**: `note JOIN book`，时间字段 `created_date`
2. **阅读计时查询**: `read_time_record JOIN book`，时间字段 `CASE WHEN fuzzy_read_date != 0 THEN fuzzy_read_date ELSE start_time END`
3. **状态变更查询**: `book_read_status_record JOIN book JOIN read_status`，时间字段 `changed_date`
4. **打卡查询**: `check_in_record JOIN book`，时间字段 `checkin_date`
5. **书评查询**: `review JOIN book`，时间字段 `created_date`
6. **相关内容/书籍查询**: `category_content JOIN book LEFT JOIN category`，按 `content_book_id` 区分相关内容 vs 相关书籍，时间字段 `created_date`
7. 所有查询 WHERE `is_deleted = 0`
8. 合并后按 `timestamp` 降序排序
9. 按 `Calendar.current.startOfDay(for:)` 分组为 `[TimelineSection]`

#### 修改 `xmnote/Data/Repositories/RepositoryContainer.swift`

新增 `timelineRepository: any TimelineRepositoryProtocol` 属性。

**参考模式：** `StatisticsRepository.swift` 的 GRDB 多表聚合查询写法。

### Phase 3: TimelineViewModel

**目标：** 状态管理、日期选择、分类过滤。

#### 新建 `xmnote/ViewModels/Reading/TimelineViewModel.swift`

```swift
@Observable
final class TimelineViewModel {
    var sections: [TimelineSection] = []
    var selectedCategory: TimelineEventCategory = .all
    var selectedDate: Date = Date()
    var isLoading = false

    private let repository: any TimelineRepositoryProtocol
}
```

核心方法：
- `loadTimeline()` — 根据 selectedDate 和 selectedCategory 查询数据
- `changeCategory(_:)` — 切换分类并重新加载
- `selectDate(_:)` — 选择日期并重新加载

### Phase 5: 时间线装饰器组件

**目标：** 实现左侧日期圆圈 + 竖线 + Emoji 的时间线视觉效果。

#### 新建 `xmnote/Views/Reading/Timeline/Components/` 下的组件

| 文件 | 职责 |
|------|------|
| `TimelineDateBadge.swift` | 日期圆圈（蓝色圆形背景 + 日期数字），同日首项显示 |
| `TimelineConnector.swift` | 竖线连接器（日期圆圈之间的连接线） |
| `TimelineEventRow.swift` | 单事件行布局（左侧装饰列 + 右侧内容卡片），包含 Emoji 图标 |

Emoji 映射（对齐 Android）：
- 书摘 📝 / 阅读计时 ⌛️ / 打卡 📅 / 书评 ✏️ / 相关内容 🗂
- 状态变更：想读 😍 / 在读 👁 / 已读 🎊 / 弃读 😵‍ / 搁置 📦

### Phase 6: 7 种事件卡片

**目标：** 实现每种事件类型的卡片 UI。

#### 新建 `xmnote/Views/Reading/Timeline/Components/` 下的卡片

| 文件 | 内容 | 关键元素 |
|------|------|---------|
| `TimelineNoteCard.swift` | 书摘卡片 | 笔记内容 + 想法 + 书名标签 |
| `TimelineTimingCard.swift` | 阅读计时卡片 | 封面(XMBookCover) + 书名/作者 + 时长徽章 |
| `TimelineStatusCard.swift` | 状态变更卡片 | 封面 + 书名/作者 + 状态标签 + 评分(已读时) |
| `TimelineCheckInCard.swift` | 打卡卡片 | 封面 + 书名/作者 + 阅读量级别徽章 |
| `TimelineReviewCard.swift` | 书评卡片 | 书评标题 + 内容 + 评分 |
| `TimelineRelevantCard.swift` | 相关内容卡片 | 标题 + 内容 + 链接 + 分类标签 |
| `TimelineRelevantBookCard.swift` | 相关书籍卡片 | 封面 + 书名/作者 + 分类标签 |

封面渲染统一使用 `XMBookCover`。间距/圆角使用 `Spacing` / `CornerRadius` token。
