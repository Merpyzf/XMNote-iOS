# 时间线功能需求文档

## 功能定位

iOS 端在读 Tab 的"时间线"子页，对齐 Android 端完整时间线功能。

## 核心需求

**多源 UNION + 时间排序 + 按日分组** 的聚合列表，左侧日期圆圈+竖线的时间线装饰。

### 7 种事件类型

| 事件类型 | 数据来源 | 时间字段 | Emoji |
|---------|---------|---------|-------|
| 书摘 | `note JOIN book` | `created_date` | 📝 |
| 阅读计时 | `read_time_record JOIN book` | `CASE WHEN fuzzy_read_date != 0 THEN fuzzy_read_date ELSE start_time END` | ⌛️ |
| 状态变更 | `book_read_status_record JOIN book JOIN read_status` | `changed_date` | 想读😍/在读👁/已读🎊/弃读😵‍/搁置📦 |
| 打卡 | `check_in_record JOIN book` | `checkin_date` | 📅 |
| 书评 | `review JOIN book` | `created_date` | ✏️ |
| 相关内容 | `category_content JOIN book LEFT JOIN category` | `created_date` | 🗂 |
| 相关书籍 | `category_content JOIN book LEFT JOIN category` | `created_date` | 🗂 |

### 日历选择器

- 月份/日期显示 + "今天"/"昨天"/"N天前" 描述
- 日期选中后触发列表重新加载
- "返回今天" 按钮

### 分类过滤

- 全部/书摘/状态/相关/书评/计时/打卡
- 切换分类后重新加载

### 默认数据范围

- 选中"今天"时加载过去 31 天
- 选中其他日期时加载该天全天数据

## 所有查询 WHERE 约束

- `is_deleted = 0`

## 相关内容 vs 相关书籍区分

- `category_content` 表中 `content_book_id` 非空时为相关书籍，否则为相关内容
