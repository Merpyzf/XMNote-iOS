# 时间线功能需求文档

## 功能定位
- 时间线是 iOS 端“在读”Tab 的正式二级页面，对齐 Android 端统计容器中的 `TimelineFragment` 主流程。
- 页面目标不是展示单一事件流，而是把阅读行为、书摘、书评、打卡、相关内容统一折叠到同一条按日分组的历史时间轴中。

## 当前交付范围

### 1. 入口与主流程
- 在读容器提供 `在读 / 时间线 / 统计` 三段切换。
- 切换到“时间线”后进入 `ReadingTimelineView` 正式页面，不再使用 placeholder。
- 页面默认加载“今天”对应的数据范围与当月日历标记。

### 2. 多源聚合列表
- 按时间倒序聚合以下 7 类事件。
- 按自然日分组渲染，并在列表中显示粘性日期头。
- 所有查询统一受 `is_deleted = 0` 约束。

| 事件类型 | iOS 数据来源 | 时间字段 | 语义标记 |
| --- | --- | --- | --- |
| 书摘 | `note JOIN book` | `created_date` | `📝` |
| 阅读计时 | `read_time_record JOIN book` | `CASE WHEN fuzzy_read_date != 0 THEN fuzzy_read_date ELSE start_time END` | `⌛️` |
| 状态变更 | `book_read_status_record JOIN book` | `changed_date` | `😍 / 👁 / 🎊 / 😵‍💫 / 📦` |
| 打卡 | `check_in_record JOIN book` | `checkin_date` | `📅` |
| 书评 | `review JOIN book` | `created_date` | `✏️` |
| 相关内容 | `category_content JOIN book LEFT JOIN category` | `created_date` | `🗂` |
| 相关书籍 | `category_content JOIN book LEFT JOIN category` | `created_date` | `🗂` |

### 3. 日历驱动浏览
- 顶部显示当前展示月份，支持菜单跳月。
- 支持横向分页切月。
- 支持点选日期后重查该日数据。
- 支持一键“回到今天”。
- 日历 cell 需要显示“该日是否活跃”和“阅读进度”标记。

### 4. 分类过滤
- 支持 `全部 / 书摘 / 状态 / 相关 / 书评 / 计时 / 打卡`。
- 切换分类后，列表和日历标记都必须同步刷新。

### 5. 数据范围策略
- 当选中日期是“今天”时，按用户时间范围配置回溯查询。
- 当前 iOS 侧已实现配置读取逻辑：`0=当天`、`1=过去 31 天`、`2=过去半年`、`3=过去一年`、`4=全部`。
- 当前默认值为 `2`，即过去半年（183 天）。
- 当选中非今天日期时，仅查询该自然日 `00:00:00 ~ 23:59:59.999`。

### 6. 相关内容与相关书籍区分
- `category_content.content_book_id != 0` 时视为“相关书籍”。
- 其余记录视为“相关内容”。
- 两者在 UI 上保持不同卡片语义：
  - 相关内容：标题 / 正文 / 图片 / 链接 / 分类标签
  - 相关书籍：被关联书籍封面 / 书名 / 作者 / 分类标签

### 7. 富文本展示策略
- 时间线中的书摘、书评、相关内容正文使用 `ExpandableRichText`。
- 收起态最多显示 `3` 行，使用系统级尾部省略号。
- 内容超出时显示“展开”，展开后显示完整富文本和“收起”。
- 收起态优先保证列表滚动流畅度，不要求保留引用竖线与自定义列表圆点。

## 当前未纳入本轮交付
- 时间范围设置入口 UI。
- 列表 item 点击进入详情页或编辑页。
- 复制、删除、TTS 等二级操作菜单。
- Android 端已有的更完整上下文操作链路。

## 需求边界
- Repository 是唯一数据入口，ViewModel 不直接访问数据库。
- `ReadingTimelineView` 负责主流程承接，不对外暴露 item 级回调。
- 页面优化目标是提升滚动流畅度，不引入额外可感知的 UI 风格变化。
