# 我的与数据备份界面精修 iOS ↔ Android 对齐情况

评估日期：2026-03-17

## 功能点清单

1. 我的页分组化设置入口保留
- 功能点：个人中心继续承载阅读设置、数据入口、管理入口、支持入口。
- iOS 行为：`PersonalView` 通过 `readingAndDataSection / managementSection / supportAndAboutSection` 继续承载原有入口集合，未减少业务入口。
- Android 行为：`PersonalContainerFragment` 继续在个人页注册同类入口点击事件（阅读日历、备份、导入导出、管理项、关于等）。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/Views/Personal/PersonalView.swift:50`、`xmnote/Views/Personal/PersonalView.swift:122`、`xmnote/Views/Personal/PersonalView.swift:140`、`xmnote/Views/Personal/PersonalView.swift:152`

2. 我的页行级点击热区与分隔语义
- 功能点：设置列表项保持可点击热区与分隔边界。
- iOS 行为：每行使用 `frame(minHeight: 44)` 保证热区，使用 `PersonalSettingsDivider` 保持分隔结构。
- Android 行为：每行使用 `paddingTop/paddingBottom=18dp` 与 item divider 形成可点击区与分隔。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/Views/Personal/PersonalView.swift:281`、`xmnote/Views/Personal/PersonalView.swift:305`

3. 云备份方式切换能力
- 功能点：支持 WebDAV 与阿里云盘两种云备份方式切换。
- iOS 行为：`providerSelectionMenu` 提供 `webdav / aliyunDrive` 两选项并调用 `selectProvider` 持久化。
- Android 行为：点击 `currCloudBackupServiceContainer` 后使用 `CascadePopupMenu` 切换 WebDAV/阿里云盘并写入设置。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/Views/Personal/Backup/DataBackupView.swift:156`、`xmnote/ViewModels/Personal/Backup/DataBackupViewModel.swift:194`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/backup/activity/BackupActivity.kt:150`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/backup/activity/BackupActivity.kt:166`

4. 空字段处理（WebDAV 未配置）
- 功能点：WebDAV 未配置时需有明确兜底文案。
- iOS 行为：provider 摘要与明细均以“未配置”作为空字段兜底，使用次级文本样式显示。
- Android 行为：当服务器 `id == 0` 时显示“未配置备份服务器”并隐藏后续可执行备份能力。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/ViewModels/Personal/Backup/DataBackupViewModel.swift:153`、`xmnote/Views/Personal/Backup/DataBackupView.swift:228`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/backup/activity/BackupActivity.kt:427`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/backup/activity/BackupActivity.kt:431`

5. 阿里云盘选中态下的冗余状态文案处理
- 功能点：选中阿里云盘时，避免重复状态噪音。
- iOS 行为：`selectedProviderSummary` 在阿里云盘场景返回 `nil`，不再在“云备份方式”行底部追加“未登录”；未登录状态由阿里云授权行承接。
- Android 行为：阿里云未授权时通过 `rlAuthAliyunpan` 单行表达，未在“云备份方式”标题下额外重复状态。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/ViewModels/Personal/Backup/DataBackupViewModel.swift:148`、`xmnote/Views/Personal/Backup/DataBackupView.swift:131`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/backup/activity/BackupActivity.kt:557`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/res/layout/activity_backup.xml:206`

6. 备份方式切换触发区域
- 功能点：点击备份方式后展示可选 provider。
- iOS 行为：仅右侧值区（provider 名称 + 箭头）参与交互过渡，左侧标题区保持稳定。
- Android 行为：整个 `currCloudBackupServiceContainer` 行均为触发区域。
- 对齐状态：有意差异。
- 双端代码证据：
  - iOS：`xmnote/Views/Personal/Backup/DataBackupView.swift:124`、`xmnote/Views/Personal/Backup/DataBackupView.swift:150`、`xmnote/Views/Personal/Backup/DataBackupView.swift:156`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/backup/activity/BackupActivity.kt:150`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/res/layout/activity_backup.xml:168`
- 评估结论：功能优化。
- Android 反向建议：
  - 是否吸收：建议吸收。
  - 建议优先级：中。
  - 风险：低。仅涉及交互触发面重构，不改变 provider 切换业务逻辑。

7. 顶部双操作按钮尺寸统一
- 功能点：首页顶部双按钮视觉尺寸一致。
- iOS 行为：`TopBarActionIcon` 增加 `containerSize`，在「我的」「笔记」页设置为 `36` 以对齐 `+` 按钮容器。
- Android 行为：个人页顶部两个 `ImageButton` 都以 `24dp` 图标尺寸嵌在同一 actionBar 高度内，尺寸天然一致。
- 对齐状态：已对齐。
- 双端代码证据：

## 总结
- 本轮以视觉与交互精修为主，未改变核心业务链路。
- 与 Android 的主要差异集中在“备份方式切换触发区域”这一交互表达，属于 iOS 端有意优化，不构成语义偏差。
