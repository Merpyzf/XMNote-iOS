# 我的与数据备份界面精修 - 设计文档

## 设计目标
- 让页面从“功能堆叠感”转为“信息分层清晰、节奏克制”。
- 在不动业务数据流的前提下，通过布局与交互细节优化提升精致度。
- 保持 Android 业务意图一致，同时使用 iOS 原生交互表达。

## 实际实现

### 1. 我的页面（PersonalView）
- 文件：`xmnote/Views/Personal/PersonalView.swift`
- 关键调整：
  - 用 `Layout.panelSpacing = Spacing.comfortable` 收敛分组间距。
  - 在分组容器增加 `panelEdgeVerticalInset`，并将行高改为 `frame(minHeight: 44)`，消除相邻 Item 的垂直 padding 叠加。
  - 移除 Section 标题，保留四组卡片分区（会员、阅读与数据、管理、支持与关于）。
  - 行内图标改为 `Color.textPrimary`，与标题文本同色，统一权重。

### 2. 数据备份页面（DataBackupView + DataBackupViewModel）
- 文件：
  - `xmnote/Views/Personal/Backup/DataBackupView.swift`
  - `xmnote/ViewModels/Personal/Backup/DataBackupViewModel.swift`
- 关键调整：
  - 云备份方式切换入口重构为“右侧值区 Menu”，左侧标题区保持稳定。
  - 切换中状态仅作用于右侧值区，避免整行视觉跳变。
  - `selectedProviderSummary` 对阿里云盘返回 `nil`，避免“已选阿里云盘 + 未登录”重复表达。
  - WebDAV 行与阿里云登录行移除冗余前置图标，突出主信息与次要说明。
  - provider 行纵向间距改为 `Spacing.cozy`，恢复与其它行一致的节奏。
  - 骨架占位（summary/detail/avatar）保持与真实内容同结构，减少加载态错位感。

### 3. 顶部按钮尺寸统一（TopBarActionIcon + NoteContainerView）
- 文件：
  - `xmnote/UIComponents/TopBar/TopBarActionIcon.swift`
  - `xmnote/Views/Note/NoteContainerView.swift`
  - `xmnote/Views/Personal/PersonalView.swift`
- 关键调整：
  - `TopBarActionIcon` 新增 `containerSize` 参数，默认 44。
  - 在「我的」「笔记」顶部右侧辅助按钮显式传入 `containerSize: 36`，与 `AddMenuCircleButton` 视觉尺寸对齐。

## 设计决策说明

### 1. 为什么去掉 Section 标题
- 标题在当前信息密度下提供的分组价值低于其占据的纵向空间。
- 卡片分组 + 图标语义已经足以表达信息层级，去标题可提升呼吸感。

### 2. 为什么云备份方式只让右侧值区可切换
- 旧方案把整行作为触发面会导致切换动画干扰标题与摘要，用户会感知“整行被吸走”。
- 右侧值区触发符合 iOS 设置页“标题稳定、值可变”的认知模型。

### 3. 为什么保留 Menu 而不是自定义弹层
- 本需求核心是清洁度与稳定性，不是扩展复杂选择逻辑。
- 系统 `Menu` 在可访问性、交互一致性与维护成本上更优。

## 影响范围
- UI 层：Personal、Backup、TopBar、Note。
- ViewModel 层：Backup provider 摘要策略。
- 数据层：无改动。
- 路由层：无改动。

## 验证口径
- 视觉：分组节奏、图标权重、行内层级符合目标。
- 交互：云备份方式切换仅右侧变化，切换中状态可感知。
- 状态：阿里云盘已选时不出现多余“未登录”。
- 构建：iOS 模拟器目标下编译通过。
