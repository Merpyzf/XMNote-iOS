# 我的与数据备份界面精修 - 需求文档

## 1. 背景
- 「我的」页面在旧实现中存在分区层级过重、Section 间距与行内间距叠加、呼吸感不足的问题。
- 「数据备份」页面存在视觉噪音：分割线偏重、图标语义干扰、备份方式切换交互脏乱、阿里云盘状态文案冗余。
- 首页顶部在出现双按钮时，`+` 按钮与旁侧按钮容器尺寸不一致，影响整体节奏。

## 2. 目标
- 统一「我的」与「数据备份」的设计语言：更轻、更干净、信息层级更明确。
- 保持功能链路不变，仅优化信息表达与交互反馈。
- 收敛顶部操作按钮尺寸，保证首页多入口场景的视觉一致性。

## 3. 范围
- 范围内：
  - 「我的」页面分组结构、间距、图标与文本样式优化。
  - 「数据备份」页面云备份方式切换行、空字段展示、阿里云授权状态表达优化。
  - 顶部通用操作图标容器尺寸能力补齐，并在「我的」「笔记」页面落地。
- 范围外：
  - 不改变备份仓储逻辑与授权流程语义。
  - 不新增备份提供方。
  - 不调整路由结构与业务入口数量。

## 4. 需求细则
- 「我的」页面：
  - 移除 Section 标题，改为纯卡片分组结构。
  - 行内图标颜色与文本颜色一致，避免视觉抢占。
  - 使用最小行高控制点击热区，避免相邻 Item 垂直 padding 叠加导致节奏失衡。
  - 缩小 Section 间距，提升纵向连续性。
- 「数据备份」页面：
  - 云备份方式切换时，只允许右侧值区参与交互反馈。
  - 切换控件采用系统菜单能力，避免整行“被吸入菜单”的错觉。
  - 选中阿里云盘时，移除冗余“未登录”底部状态文案。
  - WebDAV/阿里云授权入口前图标弱化处理，突出文本信息。
  - 骨架占位与真实内容结构保持一致。
- 顶部按钮：
  - 顶部右侧双按钮容器尺寸一致（与 `+` 按钮视觉对齐）。

## 5. 验收标准
- 「我的」页面分组间距与行内节奏稳定，不出现上紧下松的视觉错位。
- 「数据备份」页面切换云备份方式时，左侧标题区不抖动、不空白。
- 选择阿里云盘后不再出现多余“未登录”状态。
- 「我的」与「笔记」页面顶部双按钮尺寸一致。
- 编译通过：`xcodebuild -project XMNote.xcodeproj -scheme xmnote -destination 'platform=iOS Simulator,name=iPhone 17 Pro' build`。
