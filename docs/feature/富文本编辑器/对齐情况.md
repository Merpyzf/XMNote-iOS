# 富文本编辑器 iOS ↔ Android 对齐情况

评估日期：2026-03-01

## 功能点清单

1. 富文本 HTML 双向兼容链路
- 功能点：HTML 与富文本对象的双向转换。
- iOS 行为：`HTMLParser` 负责 HTML → `NSAttributedString`，`HTMLSerializer` 负责 `NSAttributedString` → HTML。
- Android 行为：`StringExtensions.htmlToSpanned` 与 `KnifeTextExtensions.html()` 提供双向转换。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/RichTextEditor/HTMLParser.swift:22`、`xmnote/RichTextEditor/HTMLSerializer.swift:26`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/common/src/main/java/com/merpyzf/common/utils/extensions/StringExtensions.kt:53`、`/Users/wangke/Workspace/AndroidProjects/XMNote/common/src/main/java/com/merpyzf/common/utils/extensions/KnifeTextExtensions.kt:22`

2. 高亮色 Int32 互转与持久化格式
- 功能点：高亮色按 Android `Int32` 写入 HTML，保证跨端读写一致。
- iOS 行为：`HighlightColors` 提供 `argbFromAndroidInt/androidInt`；序列化输出 `<mark style="background-color:{int32}">`。
- Android 行为：Toolbar 内维护高亮色板，最终通过 Knife HTML 出口落库。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/RichTextEditor/HighlightColors.swift:83`、`xmnote/RichTextEditor/HTMLSerializer.swift:235`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/common/src/main/java/com/merpyzf/common/widget/TextEditToolbarView.kt:76`、`/Users/wangke/Workspace/AndroidProjects/XMNote/common/src/main/java/com/merpyzf/common/utils/extensions/KnifeTextExtensions.kt:22`

3. 工具栏核心编辑能力
- 功能点：格式按钮、撤销/重做、清除格式、键盘收起。
- iOS 行为：`RichTextToolbar` 内置 `UNDO/REDO/FORMAT_CLEAR` 与收起键盘，`RichTextEditorView` 提供格式切换和清除 API。
- Android 行为：`TextEditToolbarView` 提供 `UNDO/REDO/FORMAT_CLEAR/HIGHLIGHT` 等动作，`NoteEditActivity` 消费动作执行编辑操作。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/RichTextEditor/RichTextToolbar.swift:94`、`xmnote/RichTextEditor/RichTextToolbar.swift:221`、`xmnote/RichTextEditor/RichTextEditorView.swift:184`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/common/src/main/java/com/merpyzf/common/widget/TextEditToolbarView.kt:59`、`/Users/wangke/Workspace/AndroidProjects/XMNote/common/src/main/java/com/merpyzf/common/widget/TextEditToolbarView.kt:354`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/note/activity/NoteEditActivity.kt:655`

4. 段落格式与链接编辑暴露层级
- 功能点：`bullet/blockquote/link` 的编辑入口是否在主工具栏对齐。
- iOS 行为：格式枚举已包含 `bulletList/blockquote/link`，编辑器层支持 `applyLink`。
- Android 行为：现有 `EditAction` 枚举未暴露 `bullet/blockquote/link` 按钮动作（以字符级样式为主）。
- 对齐状态：未对齐。
- 双端代码证据：
  - iOS：`xmnote/RichTextEditor/RichTextFormat.swift:19`、`xmnote/RichTextEditor/RichTextEditorView.swift:208`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/common/src/main/java/com/merpyzf/common/widget/TextEditToolbarView.kt:354`
- 评估结论：功能优化（iOS 增强）。
- Android 反向建议：
  - 是否吸收：建议吸收。
  - 建议优先级：中。
  - 风险：低（底层 HTML 已兼容，主要是交互入口与动作映射改造）。
