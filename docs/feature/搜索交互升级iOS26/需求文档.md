# 搜索交互升级 iOS26 - 需求文档

## 1. 背景
- 当前搜索 Tab 点击后会直接进入 `SearchView`。
- 目标是对齐 iOS 26 原生搜索体验：先在底部触发系统搜索框，开始输入后再进入搜索页面。

## 2. 目标
- 点击底部“搜索”按钮时，仅触发系统底部搜索框。
- 用户开始输入后，自动进入现有搜索 Tab 内容页面。
- 搜索页不再保留自定义顶部搜索栏和页面内手写输入框，避免双重入口。

## 3. 范围
- 范围内：
  - `MainTabView` 的搜索 Tab 激活行为改造。
  - `SearchView` 从“输入页”改为“结果承载页”。
- 范围外：
  - 搜索结果真实数据源接入与排序策略。
  - 搜索历史、推荐词等高级能力。

## 4. 验收标准
- 点“搜索”Tab 后不直接进入搜索页。
- 输入文本后自动打开搜索页。
- 搜索页不存在重复搜索输入 UI。
- 搜索页的书籍/笔记详情导航能力保持不变。
