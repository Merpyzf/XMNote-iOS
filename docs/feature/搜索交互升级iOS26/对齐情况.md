# 搜索交互升级 iOS26 iOS ↔ Android 对齐情况

评估日期：2026-03-01

## 功能点清单

1. 搜索入口激活机制
- 功能点：点击底部搜索入口后的首屏交互。
- iOS 行为：使用 `role: .search + .tabViewSearchActivation(.searchTabSelection) + .searchable`，优先触发系统搜索体验。
- Android 行为：通过 `SearchActivity` 进入独立搜索页面，页面内 `searchField` 直接聚焦并执行搜索。
- 对齐状态：有意差异。
- 双端代码证据：
  - iOS：`xmnote/Views/MainTabView.swift:112`、`xmnote/Views/MainTabView.swift:131`、`xmnote/Views/MainTabView.swift:132`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/search/activity/SearchActivity.kt:36`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/search/activity/SearchActivity.kt:79`
- 评估结论：功能优化（iOS 原生化表达）。
- Android 反向建议：
  - 是否吸收：可选吸收。
  - 建议优先级：低。
  - 风险：中（涉及导航语义和系统搜索入口重构）。

2. 搜索输入控件位置
- 功能点：输入框在系统层还是页面层。
- iOS 行为：`SearchView` 不再自带输入框，输入来自系统 `searchable` 绑定。
- Android 行为：`SearchActivity` 页面内持有 `searchField`，并绑定 `onSearch` 回调。
- 对齐状态：有意差异。
- 双端代码证据：
  - iOS：`xmnote/Views/MainTabView.swift:250`、`xmnote/Views/MainTabView.swift:282`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/search/activity/SearchActivity.kt:66`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/search/activity/SearchActivity.kt:79`
- 评估结论：功能优化（遵循 iOS 26 搜索交互）。
- Android 反向建议：
  - 是否吸收：按平台策略决定，不强制。
  - 建议优先级：低。
  - 风险：中（会改变 Android 用户既有输入心智）。

3. 结果容器完整度
- 功能点：搜索结果是否具备多类型分栏与真实结果承载。
- iOS 行为：当前 `SearchView` 仍以占位态文案为主，未完成分类结果分栏承载。
- Android 行为：`SearchActivity` 已接入 `ViewPager + TabLayout`，按结果数量动态生成结果页。
- 对齐状态：未对齐。
- 双端代码证据：
  - iOS：`xmnote/Views/MainTabView.swift:282`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/search/activity/SearchActivity.kt:134`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/search/activity/SearchActivity.kt:178`
- 评估结论：设计倒退（功能尚未补齐）。
- Android 反向建议：
  - 是否吸收：不建议吸收 iOS 当前占位实现。
  - 建议优先级：高（iOS 需补齐）。
  - 风险：高（直接影响搜索可用性与留存）。

4. 搜索入口上下文注入能力
- 功能点：不同业务入口传入不同搜索域与默认选中项。
- iOS 行为：当前以统一 Tab 搜索入口为主，尚无按入口注入 `searchItems` 的能力。
- Android 行为：`SearchActivity.start(...)` 支持按入口构造 `searchItems` 与默认选中项。
- 对齐状态：未对齐。
- 双端代码证据：

  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/search/activity/SearchActivity.kt:232`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/search/activity/SearchActivity.kt:292`
- 评估结论：设计倒退（业务入口语义丢失）。

  - 建议优先级：中高（iOS 应补入口上下文能力）。
  - 风险：中（影响不同场景下的搜索效率与命中率）。
