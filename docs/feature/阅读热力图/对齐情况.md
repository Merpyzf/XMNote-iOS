# 阅读热力图 iOS ↔ Android 对齐情况

评估日期：2026-03-01

## 功能点清单

1. 周列网格与星期标签固定策略
- 功能点：GitHub 风格周列网格、星期标签固定显示。
- iOS 行为：`HeatmapChart` 使用 `HStack(ScrollView + weekdayLabels)`，星期列独立于横向滚动区域。
- Android 行为：`HistoryChart` 内部维护周列网格与固定星期文案数组（`日/一/二/...`）进行绘制。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/UIComponents/Charts/HeatmapChart.swift:132`、`xmnote/UIComponents/Charts/HeatmapChart.swift:244`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/common/src/main/java/com/merpyzf/common/widget/chart/HistoryChart.kt:50`、`/Users/wangke/Workspace/AndroidProjects/XMNote/common/src/main/java/com/merpyzf/common/widget/chart/HistoryChart.kt:85`

2. 统计类型与热度阈值
- 功能点：阅读/书摘/打卡/全部 的等级阈值映射。
- iOS 行为：`HeatmapStatisticsDataType` 与 `HeatmapLevel` 按秒数与数量阈值计算五级强度。
- Android 行为：`AppConstant.HeatChartStatisticsDataType` 与 `ReadTimeColorLevel/NoteCountColorLevel` 定义同类阈值。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/Domain/Models/HeatmapModels.swift:14`、`xmnote/Domain/Models/HeatmapModels.swift:188`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/common/src/main/java/com/merpyzf/common/constant/AppConstant.java:444`、`/Users/wangke/Workspace/AndroidProjects/XMNote/common/src/main/java/com/merpyzf/common/constant/AppConstant.java:602`

3. 书籍状态色 + 阅读量色分段叠加
- 功能点：单日方格的多段配色规则。
- iOS 行为：`segmentColors(for:)` 先叠书籍状态色，再按统计类型补充阅读量色。
- Android 行为：`Mark.getColors` 先加入书籍状态色，再按统计类型追加阅读量色。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/Domain/Models/HeatmapModels.swift:128`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/common/src/main/java/com/merpyzf/common/model/vo/Mark.kt:49`

4. 方格点击跳转阅读日历
- 功能点：点击热力图单元进入阅读日历对应日期。
- iOS 行为：非未来日期点击触发 `onDayTap`，由 `ReadingHeatmapWidgetView` 上抛到阅读日历路由。
- Android 行为：`historyChart.setController` 回调直接启动 `ReadCalendarActivity` 并传时间戳。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/UIComponents/Charts/HeatmapChart.swift:602`、`xmnote/Views/Reading/ReadingHeatmapWidgetView.swift:41`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/data/fragment/components/HeatChartFragment.kt:68`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/data/fragment/components/HeatChartFragment.kt:71`

5. 帮助入口与统计类型切换承载位置
- 功能点：热力图说明与统计类型切换入口布局。
- iOS 行为：在热力图卡片中以内联 info 按钮 + Sheet 承载说明与相关控制。
- Android 行为：在 `ReadingFragment` 中通过帮助按钮弹窗，并跳转设置页进行进一步配置。
- 对齐状态：有意差异。
- 双端代码证据：
  - iOS：`xmnote/Views/Reading/ReadingHeatmapWidgetView.swift:70`、`xmnote/Views/Reading/ReadingHeatmapWidgetView.swift:104`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/main/fragment/statistics/ReadingFragment.kt:173`
- 评估结论：功能优化（iOS 降低路径层级，操作更直接）。
- Android 反向建议：
  - 是否吸收：建议吸收部分能力（至少支持“说明弹层内直接切换统计类型”）。
  - 建议优先级：中。
  - 风险：中（涉及现有设置页入口迁移和用户习惯过渡）。
