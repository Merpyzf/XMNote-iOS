# 笔记查看 对齐情况

## 1. 基线
- 评估日期：2026-03-18
- Android 基线：
  - `TimelineFragment`
  - `NoteViewActivity`
  - `ReviewViewActivity`
  - `RelevantViewActivity`
- iOS 基线：
  - `MainTabView`
  - `ContentRoute`
  - `NoteViewerView`
  - `ContentViewerView`
  - `ContentRepository`

## 2. 总体结论
- 结论：**核心业务已对齐，交互架构存在两项有意差异。**
- 已对齐部分：
  - 书摘查看支持左右滑动、分页进度、底部操作与删除后相邻项回退。
  - 时间线进入查看页后，均能按当前 source 读取有效内容并展示详情。
- 有意差异：
  - iOS 将时间线中的书摘/书评/相关内容统一收敛到混合 viewer，而 Android 的书评与相关仍单独打开。
  - iOS 当前统一只做硬删除；Android 的相关内容仍是软删除。

## 3. 功能点逐项对齐

1. 书摘全屏左右滑动
- 功能点：从时间线或书籍详情进入后，可左右滑动切换书摘。
- iOS 行为：`NoteViewerView` + `NoteViewerContentView` 使用自建 horizontal paging，并通过 `visibleNoteItems(radius:)` 与 `prefetchDetails(around:radius:)` 实现窗口化挂载与邻页预取。
- Android 行为：`NoteViewActivity` 使用 `ViewPager` 切页，并在接近尾部时触发预加载下一页。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/Views/Content/NoteViewerView.swift:67`、`xmnote/Views/Content/NoteViewerContentView.swift:73`、`xmnote/ViewModels/Content/NoteViewerViewModel.swift:165`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/note/activity/NoteViewActivity.kt:121`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/note/activity/NoteViewActivity.kt:130`

2. 书摘查看底部工具栏
- 功能点：标签、编辑、分享、删除等底部操作可在全屏查看时直接触达。
- iOS 行为：底部动作收敛到 `ImmersiveBottomChromeOverlay`，使用悬浮 ornament 风格承载标签、编辑、分享和删除。
- Android 行为：`NoteViewActivity` 底部栏提供标签、编辑、分享、删除、AI 助手等操作。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/Views/Content/NoteViewerView.swift:145`、`xmnote/UIComponents/Foundation/ImmersiveBottomChrome.swift:42`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/note/activity/NoteViewActivity.kt:150`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/note/activity/NoteViewActivity.kt:189`

3. 时间线进入混合内容查看
- 功能点：从时间线进入后，是否按当前 source 查看全部内容。
- iOS 行为：`MainTabView` 会把 `timeline(.allContent/.review/.relevant)` 路由到 `ContentViewerView`，在同一 viewer 内左右滑动查看书摘、书评、相关内容。
- Android 行为：时间线点击书摘进入 `NoteViewActivity`；点击书评进入 `ReviewViewActivity`；点击相关进入 `RelevantViewActivity`，三类内容分开打开。
- 对齐状态：有意差异。
- 双端代码证据：
  - iOS：`xmnote/Views/MainTabView.swift:237`、`xmnote/Views/MainTabView.swift:368`、`xmnote/Views/Content/ContentViewerView.swift:75`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/main/fragment/statistics/TimelineFragment.kt:332`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/main/fragment/statistics/TimelineFragment.kt:356`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/main/fragment/statistics/TimelineFragment.kt:365`
- 评估结论：功能优化。
- Android 反向建议：
  - 是否吸收：建议吸收。
  - 建议优先级：中高。
  - 风险：中。需要重构书评/相关查看器路由与数据组织，但能显著提升时间线浏览连续性。

4. 书评与相关内容横向滑动架构
- 功能点：书评和相关内容查看是否也使用与阅读日历一致的自建 paging 架构。
- iOS 行为：`ContentViewerContentView` 使用 `ScrollView(.horizontal) + LazyHStack + .scrollTargetBehavior(.paging) + .scrollPosition`，不再使用 `TabView`。
- Android 行为：书评和相关内容各自单页 Activity 打开，不存在横向滑动容器。
- 对齐状态：有意差异。
- 双端代码证据：
  - iOS：`xmnote/Views/Content/ContentViewerContentView.swift:77`、`xmnote/ViewModels/Content/ContentViewerViewModel.swift:170`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/note/fragment/ReviewViewActivity.kt:139`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/note/fragment/RelevantViewActivity.kt:139`
- 评估结论：功能优化。
- Android 反向建议：
  - 是否吸收：建议吸收。
  - 建议优先级：中。
  - 风险：中。需要先统一书评/相关内容的数据 feed 和 page state 抽象。

5. 删除语义
- 功能点：查看页删除当前内容时的数据删除方式。
- iOS 行为：`ContentRepository.delete(itemID:)` 对书摘、书评、相关内容统一执行硬删除事务；查看页文案也明确提示“iOS 端当前按硬删除实现”。
- Android 行为：书评仓储执行硬删除；相关内容仓储仍以软删除语义为主。
- 对齐状态：有意差异。
- 双端代码证据：
  - iOS：`xmnote/Data/Repositories/ContentRepository.swift:122`、`xmnote/Views/Content/ContentViewerView.swift:117`、`xmnote/Views/Content/NoteViewerView.swift:107`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/data/src/main/java/com/merpyzf/data/repository/ReviewRepository.kt:70`、`/Users/wangke/Workspace/AndroidProjects/XMNote/data/src/main/java/com/merpyzf/data/repository/RelevantRepository.kt:259`
- 评估结论：功能优化。
- Android 反向建议：
  - 是否吸收：暂不建议直接吸收。
  - 建议优先级：低。
  - 风险：高。该项受数据库语义、同步链路和回收站策略影响，不能只改 UI 行为。

6. 删除后相邻项回退
- 功能点：删除当前查看内容后是否自动切到相邻项。
- iOS 行为：`NoteViewerViewModel` 与 `ContentViewerViewModel` 都记录删除前索引，删除成功后自动回退到相邻项；若 source 已空则请求关闭页面。
- Android 行为：`NoteViewActivity` 删除书摘后会重建 adapter 并回退到当前可用位置；书评/相关删除后直接关闭单页查看。
- 对齐状态：部分对齐。
- 双端代码证据：
  - iOS：`xmnote/ViewModels/Content/NoteViewerViewModel.swift:130`、`xmnote/ViewModels/Content/ContentViewerViewModel.swift:132`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/note/activity/NoteViewActivity.kt:195`

## 4. 结论建议
- iOS 当前实现已经完成“书摘业务意图对齐 + 统一查看架构 + 更沉浸的原生表达”。
- 若 Android 后续要跟进，可以优先吸收两项改造：
  - 时间线进入统一混合 viewer。
  - 书评/相关内容切换到与书摘一致的分页架构。
- 删除语义暂不建议 Android 跟随 iOS 直接改为硬删除，应该由数据库与同步策略专项评估后再决策。
