# 图片加载管线迁移（Nuke）- 设计文档

## 1. 设计目标
- 构建统一图片加载底座，避免页面层重复实现网络下载逻辑。
- 保证 UI 展示与数据层下载共享同一请求策略与缓存策略。
- 在不破坏现有 MVVM/Repository 边界的前提下完成替换。

## 2. 分层架构

### 2.1 Infra：图片管线基础设施
- `XMImagePipelineFactory`
  - 负责构建默认 `ImagePipeline` 配置（超时、缓存、并发等）。
- `XMImageRequestBuilder`
  - 负责 URL 归一化、请求头、防盗链、优先级、GIF 探测规则。
- `XMCoverImageLoading` + `NukeCoverImageLoader`
  - 为 Data 层提供“与 UI 解耦”的图片下载抽象。

### 2.2 UIComponents：统一图片组件
- `XMRemoteImage`
  - 封装 `LazyImage` 静态图路径。
  - 处理 GIF URL、响应头与数据探测。
  - 失败自动降级占位或静态图。
- `XMGIFImageView`
  - 以 `UIViewRepresentable` 桥接 Gifu，承担 GIF 播放生命周期管理。

### 2.3 Data：业务仓储接入
- `ReadCalendarColorRepository`
  - 依赖 `XMCoverImageLoading` 获取封面图，再执行颜色提取与缓存。
- `RepositoryContainer`
  - 统一注入 `coverImageLoader`，避免仓储自行 new 具体实现。

### 2.4 App 启动编排
- `xmnoteApp`
  - 在初始化阶段设置 `ImagePipeline.shared = XMImagePipelineFactory.makeDefault()`。

## 3. 关键数据流

### 3.1 UI 静态图路径
1. 页面调用 `XMRemoteImage(urlString: ...)`。
2. `XMImageRequestBuilder.makeImageRequest` 生成统一请求。
3. `LazyImage` 走 `ImagePipeline.shared` 下载/缓存/解码。
4. 成功渲染；失败显示 placeholder。

### 3.2 UI GIF 路径
1. `XMRemoteImage` 判断 URL/响应头/数据签名是否 GIF。
2. 命中 GIF 时加载二进制 `Data`。
3. `XMGIFImageView`（Gifu）播放动画。
4. 失败降级静态图或 placeholder。

### 3.3 Data 取色路径
1. `ReadCalendarColorRepository` 请求 `XMCoverImageLoading.loadImage(...)`。
2. 下载成功后提取封面主色。
3. 提取失败或下载失败使用哈希回退色并缓存。

## 4. 调试与观测
- `ImageLoadingTestViewModel` 直接基于 `ImagePipeline` 执行探测请求，采集：
  - 成功/失败状态
  - 耗时
  - 缓存来源（memory/disk/network）
- `ImageLoadingTestView` 可视化输出批量样例与手动 URL 测试结果。

## 5. 接口与约束
- 新增公开复用组件：
  - `XMRemoteImage<Placeholder: View>`
  - `XMGIFImageView`
  - `ReadingDurationRankingChart`（本轮与封面加载联动更新）
- 新增基础设施：
  - `XMImagePipelineFactory`
  - `XMImageRequestBuilder`
  - `XMCoverImageLoading`
  - `NukeCoverImageLoader`

## 6. 异常与降级策略
- URL 非法：直接 placeholder。
- 网络失败：显示 placeholder，测试页输出错误信息。
- GIF 探测失败：降级静态图。
- 取色失败：回退默认色，禁止长期 pending。

## 7. 验证策略
- 编译验证：
  - `xcodebuild -project XMNote.xcodeproj -scheme xmnote -destination 'platform=iOS Simulator,name=iPhone 17 Pro' build`
- 调试验证：
  - 通过 DebugCenter -> 图片加载页执行预置样例，确认 GIF、404、超时、缓存来源链路。
