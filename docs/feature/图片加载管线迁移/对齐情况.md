# 图片加载管线迁移 iOS ↔ Android 对齐情况

评估日期：2026-03-03

## 功能点清单

1. 防盗链请求头统一
- 功能点：网络图片请求是否统一注入 `User-Agent` / `Referer`。
- iOS 行为：`XMImageRequestBuilder` 统一设置 `User-Agent`，豆瓣域名追加 `Referer`。
- Android 行为：`GlideImageLoader.glideUrl(...)` 统一设置 `User-Agent`，豆瓣域名追加 `Referer`。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/Infra/ImagePipeline/XMImageRequestBuilder.swift:110`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/common/src/main/java/com/merpyzf/common/service/imageloader/GlideImageLoader.kt:173`

2. 页面层统一图片加载封装
- 功能点：页面是否通过统一组件/封装加载网络图片，而非散落实现。
- iOS 行为：`XMRemoteImage` 作为统一入口，`BookGridItemView`、`BookDetailView`、`ReadingDurationRankingChart` 均复用该组件。
- Android 行为：页面层主路径通过 `GlideImageLoader.loadFromUrl(...)` 统一封装调用。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/UIComponents/Foundation/XMRemoteImage.swift:12`、`xmnote/Views/Book/BookGridItemView.swift:30`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/common/src/main/java/com/merpyzf/common/service/imageloader/GlideImageLoader.kt:58`

3. GIF 动图处理细节
- 功能点：GIF 识别、播放与用户提示策略。
- iOS 行为：`XMRemoteImage` 支持 URL/响应头/二进制三段探测 GIF，命中后使用 `XMGIFImageView` 播放，并可显示 `GIF` 角标。
- Android 行为：`Glide` 主路径可加载 GIF，但未在阅读主链路提供 GIF 角标与“静态图后探测切换 GIF”策略。
- 对齐状态：有意差异。
- 双端代码证据：
  - iOS：`xmnote/UIComponents/Foundation/XMRemoteImage.swift:45`、`xmnote/UIComponents/Foundation/XMGIFImageView.swift:12`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/common/src/main/java/com/merpyzf/common/service/imageloader/GlideImageLoader.kt:77`
- 评估结论：功能优化。
- Android 反向建议：
  - 是否吸收：建议按业务需要吸收（尤其是书封面源不稳定场景）。
  - 建议优先级：中。
  - 风险：低中（需要补充 GIF 标识与降级策略）。

4. 非 UI 场景下载抽象（取色链路）
- 功能点：数据层取色下载是否复用统一图片加载抽象。
- iOS 行为：`ReadCalendarColorRepository` 依赖 `XMCoverImageLoading`，由 `RepositoryContainer` 注入 `NukeCoverImageLoader`，UI 与 Data 共用同一请求策略。
- Android 行为：`ReadCalendarRepository` 在仓储内直接调用 `PaletteHelper.getPalette(...)` 取色，未通过独立下载抽象层解耦。
- 对齐状态：未对齐。
- 双端代码证据：
  - iOS：`xmnote/Data/Repositories/ReadCalendarColorRepository.swift:12`、`xmnote/Data/Repositories/RepositoryContainer.swift:32`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/data/src/main/java/com/merpyzf/data/repository/ReadCalendarRepository.kt:298`
- 评估结论：功能优化。
- Android 反向建议：
  - 是否吸收：建议吸收（可降低取色链路与具体库耦合）。
  - 建议优先级：中高。
  - 风险：中（涉及仓储层接口调整与回归验证）。
