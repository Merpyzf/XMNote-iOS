# 书摘编辑体验优化 iOS ↔ Android 对齐情况

评估日期：2026-03-27

## 功能点清单

1. 聚焦摘录模式下，想法收起时摘录高度分配
- 功能点：在“想法收起”状态优先保证摘录输入区域可编辑与可见。
- iOS 行为：`focus.collapsed` 分支按键盘状态切换“可视区优先/可用区优先”，并使用统一计算入口输出高度。
- Android 行为：`calculateAndApplyContentHeight` 按键盘状态切换 `contentAvailableToFitAll` 与 `fullViewportHeight` 策略。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/Views/Note/NoteEditorView.swift:1441-1529`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/note/activity/NoteEditActivity.kt:1189-1233`

2. 固定区高度参与编辑区计算（书籍信息/元数据/附件/OCR）
- 功能点：高度分配时排除固定元素，避免输入区与元数据互相挤压。
- iOS 行为：`fixedElementsHeight = book + stableTail + padding + gaps`，固定区来源于实际测量，并允许种子值兜底。
- Android 行为：`calculateFixedElementsHeight()` 显式累加书籍、页码、章节、标签、创建时间、分割线、间距、附件与 OCR 区。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/Views/Note/NoteEditorView.swift:126-131`、`1452-1459`、`1532-1544`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/note/activity/NoteEditActivity.kt:1329-1375`

3. 元数据行结构（位置/章节/标签/创建时间）
- 功能点：元数据区为连续行式编辑入口，统一行高与右侧操作 affordance。
- iOS 行为：统一通过 `metadataActionRow` 构建章节/标签/创建时间行，右附件槽位固定 `12pt`，标签空态右对齐。
- Android 行为：章节、标签、创建时间分别是独立 `RelativeLayout/LinearLayout`，右侧附件为 `24dp` 图标或按钮。
- 对齐状态：有意差异。
- 双端代码证据：
  - iOS：`xmnote/Views/Note/NoteEditorView.swift:835-931`、`944-960`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/res/layout/activity_note_edit.xml:283-442`
- 评估结论：功能优化（iOS 可维护性与视觉一致性提升）。
- Android 反向建议：
  - 是否吸收：建议吸收“统一行构建器/统一附件槽位”思想；
  - 建议优先级：中；
  - 风险：低（主要是布局重构，不涉及数据语义变更）。

4. 标签空态交互文案
- 功能点：标签未选择时的可操作提示。
- iOS 行为：显示“添加标签”空态文案，右对齐并保留右侧箭头。
- Android 行为：以 `btnAddTag` 图标按钮为主，无“添加标签”文本空态。
- 对齐状态：有意差异。
- 双端代码证据：
  - iOS：`xmnote/Views/Note/NoteEditorView.swift:877-903`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/res/layout/activity_note_edit.xml:373-392`
- 评估结论：功能优化（iOS 在空态可发现性更强）。
- Android 反向建议：
  - 是否吸收：可选吸收；
  - 建议优先级：低；
  - 风险：低（仅文案与布局层改动）。

5. 创建时间自动走时与手动接管
- 功能点：新建书摘默认跟随系统时间；用户手动选时间后停止自动更新。
- iOS 行为：`syncCreatedDateAutoUpdateState` + `start/stopCreatedDateAutoUpdateIfNeeded` 控制自动更新生命周期。
- Android 行为：`startCreateTimeAutoUpdate` 定时更新，`onPickDateTime` 中调用 `stopCreateTimeAutoUpdate` 停止自动更新。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/ViewModels/Note/NoteEditorViewModel.swift:624-657`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/note/activity/NoteEditActivity.kt:172-188`、`1708-1714`

6. 字体层级治理（正文/元数据标题/元数据值）
- 功能点：避免编辑页字体入口分散导致观感混乱。
- iOS 行为：正文与元数据文本统一走 `AppTypography`，标题和值分层明确。
- Android 行为：元数据和编辑输入主要使用 `@dimen/sub_medium_text`，层级较扁平但风格一致。
- 对齐状态：部分对齐（语义层级表达有差异）。
- 双端代码证据：
  - iOS：`xmnote/Views/Note/NoteEditorView.swift:133-138`、`933-942`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/res/layout/activity_note_edit.xml:133`、`177`、`303`、`371`、`417`
- 评估结论：功能优化（iOS 语义化层级更细，便于后续动态字体治理）。
- Android 反向建议：
  - 是否吸收：建议吸收“标题/值/正文”三层 token；
  - 建议优先级：中；
  - 风险：中（需联动全页字号回归验证）。
