# 书摘编辑体验优化 — 设计文档

## 1. 问题分层

### L1 现象层

- 聚焦摘录模式初次进入时高度异常，输入一个字符后高度才“恢复正常”；
- 元数据区多行布局看起来一致，但实现是多段复制，后续每次改间距都需要逐行打补丁。

### L2 机制层

- 高度计算依赖测量值，但首次渲染阶段测量尚未完全稳定；
- 调试探针与业务链路耦合，阅读和修改时容易误判真实入口；
- 元数据行缺少统一构建器，标题/值/附件槽位靠手工拼接维持一致。

### L3 方案层

1. 保留单一高度计算入口，收敛为“输入参数 -> 计算结果对象”；
2. 调试日志通过统一代理入口接入，避免业务节点直接依赖 Debug 方法；
3. 元数据行抽象为通用行构建器，统一 spacing、rowHeight、trailing accessory width。

## 2. 核心设计

## 2.1 聚焦模式高度计算

- 入口：`makeEditorHeightComputation(...)`  
  代码：`xmnote/Views/Note/NoteEditorView.swift:1441-1529`
- 关键输入：
  - `bookSectionHeight`
  - `stableTailSectionHeight`
  - `toolbarHeight`
  - `ideaState`
  - `effectiveKeyboardVisible`
- 关键策略：
  - `focusExcerpt + collapsed + keyboard hidden`：优先 `full_viewport_priority`，让摘录区尽可能占满可视区；
  - `focusExcerpt + collapsed + keyboard visible`：退回 `availableForEditors - collapsedIdeaHeight`，保证“想法入口可达”。

该策略与 Android 的 `calculateAndApplyContentHeight` 保持同一业务意图：  
`availableContainerHeight - fixedElements - collapsedIdeaHeight` 与全视口优先分支并存。  
Android 证据：`NoteEditActivity.kt:1189-1233`。

## 2.2 元数据行收敛

- 抽象：
  - `metadataActionRow(title:value:)`
  - `metadataActionRow(title:value:accessory:)`
  - `metadataTagValue(_:)`
- 代码：`xmnote/Views/Note/NoteEditorView.swift:835-931`
- 统一 token：
  - `metadataRowHeight = 54`（`946`）
  - `metadataRowSpacing = Spacing.cozy`（`948`）
  - `metadataTrailingAccessoryWidth = 12`（`944`）

收益：

1. 章节/标签/创建时间行的对齐规则只维护一份；
2. 标签空态与有值态共享同一行骨架；
3. 右侧附件（箭头/清空）对齐槽位固定，避免后续逐行回归。

## 2.3 字体层级收敛

- 输入正文：`editorContentFont = AppTypography.fixed(baseSize: 16, ...)`  
  代码：`xmnote/Views/Note/NoteEditorView.swift:133-138`
- 元数据标题：`metadataTitleFont = AppTypography.fixed(baseSize: 14, weight: .medium, ...)`  
  代码：`933-940`
- 元数据值：`metadataValueFont = AppTypography.subheadline`  
  代码：`942`

设计目标：正文/标题/值的角色边界固定，减少“局部调大/调小”造成的视觉不和谐。

## 2.4 创建时间自动走时

- iOS：新建模式且未手动改时间时，每秒同步系统时间；手动选择后停止自动更新。  
  代码：`xmnote/ViewModels/Note/NoteEditorViewModel.swift:624-657`
- Android：`startCreateTimeAutoUpdate/stopCreateTimeAutoUpdate` 同样按新建与手动选择切换。  
  代码：`NoteEditActivity.kt:172-188`、`1708-1714`

## 3. 重构落地点

1. 删除未使用的高度薄封装，保留单一计算入口；
2. 新增 `observeEditorTextChange` 作为 Debug 诊断代理入口；  
   代码：`xmnote/Views/Note/NoteEditorView.swift:1435-1439`
3. 元数据三行改为统一构建器，替换重复 HStack 拼装；  
   代码：`835-873`、`906-931`
4. 保持用户已确认视觉结果不变，仅做结构收敛。
