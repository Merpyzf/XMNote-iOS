# 在读首页 iOS ↔ Android 对齐情况

评估日期：2026-03-11

## 功能点清单

1. 首页真实内容入口
- 功能点：在读 Tab 首页提供真实阅读概览，而不是占位页。
- iOS 行为：`ReadingContainerView` 已在首页子页挂载 `ReadingDashboardView`，页面进入后创建 `ReadingDashboardViewModel` 并启动 observation。
- Android 行为：`ReadingFragment` 进入后执行 `presenter.getPageData()`，拉取页面所需概览数据。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/Views/Reading/ReadingContainerView.swift:92`、`xmnote/Views/Reading/ReadingDashboardView.swift:46`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/main/fragment/statistics/ReadingFragment.kt:255`

2. 热力图概览与进入阅读日历
- 功能点：首页顶部展示阅读热力图，并支持点击日期进入阅读日历。
- iOS 行为：`ReadingDashboardView` 顶部挂载 `ReadingHeatmapWidgetView`；点击热力图日期后通过 `onOpenReadCalendar` 交给上层路由；组件内部支持帮助入口与跨天刷新。
- Android 行为：`ReadingFragment` 初始化 `HeatChartFragment`；热力图点击后直接调用 `ReadCalendarActivity.start(...)` 打开阅读日历。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/Views/Reading/ReadingDashboardView.swift:70`、`xmnote/Views/Reading/ReadingHeatmapWidgetView.swift:42`、`xmnote/Views/Reading/ReadingHeatmapWidgetView.swift:101`

3. 趋势数据概览（三项统计 + 条形图）
- 功能点：首页展示阅读时长、书摘数量、已读书籍三项概览及近期趋势图。
- iOS 行为：`ReadingTrendMetricsSection` 使用单卡三分栏展示三项指标；`ReadingDashboardRepository.buildTrendMetrics(...)` 聚合三项主值与趋势点列。
- Android 行为：`ReadingFragment` 通过 `initReadDataCharts()` 和 `showReadDataCharts(...)` 展示阅读时长、书摘数、已读数三组图表。
- 对齐状态：已对齐。
- 双端代码证据：

4. 今日阅读目标与进度展示
- 功能点：首页展示今日阅读目标、今日实际阅读时长和目标达成状态，并支持修改目标。
- iOS 行为：`ReadingFeatureCardsSection` 内的 `ReadingDailyGoalCard` 以底部缺口弧环展示今日阅读；点击后弹出 `ReadingGoalEditorSheet`；`ReadingDashboardViewModel.saveGoal()` 通过仓储回写每日目标。

5. 继续阅读入口
- 功能点：首页展示最近一次可继续阅读的书籍，并允许快速返回阅读。
- iOS 行为：`ReadingResumeBookCard` 展示最近一次计时关联且未处于“读完/弃读”的书；无书时显示添加引导；点击后进入书籍详情或添加流程。
- Android 行为：`ReadingFragment` 点击 `resumeBookTimingContainer` 时，若无书则弹出选书 Sheet；若有书则进入 `ReadTimingActivity`。
- 对齐状态：有意差异。
- 双端代码证据：
  - iOS：`xmnote/Views/Reading/ReadingDashboardView.swift:82`、`xmnote/Views/Reading/Components/ReadingDashboardCards.swift:471`、`xmnote/Data/Repositories/ReadingDashboardRepository.swift:273`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/main/fragment/statistics/ReadingFragment.kt:233`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/mvp/contract/main/statistics/ReadingContract.kt:28`
- 评估结论：功能优化。
- Android 反向建议：
  - 是否吸收：可吸收。

6. 最近在读列表

8. 首页数据刷新方式
- 功能点：阅读记录或目标变化后，首页概览能自动回流最新结果。
- iOS 行为：`ReadingDashboardViewModel` 通过 `observeDashboard(...)` 持续观察聚合快照，目标修改后依赖 observation 自动刷新；跨天时通过 `refreshIfNeeded()` 重建订阅。
- Android 行为：`ReadingFragment` 主要依赖 `presenter.getPageData()`、`onResume()` 和局部刷新回调手动重新拉取页面数据。
- 对齐状态：有意差异。
- 双端代码证据：
  - iOS：`xmnote/ViewModels/Reading/ReadingDashboardViewModel.swift:65`、`xmnote/ViewModels/Reading/ReadingDashboardViewModel.swift:73`、`xmnote/Data/Repositories/ReadingDashboardRepository.swift:30`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/main/fragment/statistics/ReadingFragment.kt:92`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/main/fragment/statistics/ReadingFragment.kt:255`
- 评估结论：功能优化。
- Android 反向建议：
  - 是否吸收：可吸收。
  - 建议优先级：中高。
  - 风险：中。若 Android 后续也改为观察式首页聚合，需要先统一仓储层事件回流口径。

## 总结
- iOS 在读首页已经补齐 Android `ReadingFragment` 的核心概览链路，并进一步用更清晰的卡片层级和 observation 刷新模式做了原生化表达。
- 当前主要差距集中在最近在读的长按快捷操作和继续阅读直达计时，这些属于后续动作链路补齐，不属于首页信息架构倒退。
