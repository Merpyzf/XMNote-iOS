# 在读首页需求文档

## 功能定位
- 在读首页是 iOS 端“在读”Tab 的主内容区，对齐 Android 端 `ReadingFragment` 的阅读概览职责。
- 页面目标不是简单堆卡片，而是把“今日阅读状态 + 阅读趋势 + 热力图入口 + 继续阅读 + 最近在读 + 年度目标”整合为一个可快速扫读的首页仪表盘。

## 本轮交付范围

### 1. 首页壳层与入口
- `ReadingContainerView` 在首页子页挂载 `ReadingDashboardView`，替代旧的占位式在读内容。
- 页面进入后自动订阅首页聚合数据，并在应用回到前台且跨天时刷新“今日/今年”语义。

### 2. 热力图概览
- 首页顶部显示阅读热力图卡片。
- 点击任意日期后进入阅读日历。
- 右上角提供热力图说明入口。
- 热力图在跨天、阅读记录变更后应能刷新到最新结果。

### 3. 趋势数据概览
- 热力图下方展示单张三栏趋势总卡。
- 三栏指标固定为：
  - 阅读时长
  - 书摘数量
  - 已读书籍
- 每栏由以下内容组成：
  - 主值（数字使用品牌字体，单位缩小）
  - 二级标题
  - 最近窗口条形图
- 三栏不拆成三张独立卡片，不设置卡片间距，只通过分割线分隔。
- 条形图需要保留淡灰轨道，并兼容数据量级差异较大时的小值可见性。

### 4. 今日阅读与继续阅读双功能卡
- 首页中段以双列形式并排展示：
  - 今日阅读
  - 继续阅读
- 今日阅读卡需要展示：
  - 当前是否达成目标
  - 今日实际阅读时长
  - 底部缺口的弧形进度条
  - 缺口处目标时长文案
- 继续阅读卡需要展示：
  - 最近可继续的书籍
  - 当前阅读进度
  - 无书时的空态引导

### 5. 最近在读
- 横向展示最近活跃书籍列表。
- 点击书籍进入详情页。
- 列表为空时显示首页语义下的空态提示。

### 6. 年度目标与年度已读摘要
- 首页底部展示“今年已读”摘要卡。
- 点击后弹出年度已读列表。
- 支持从首页直接调整年度阅读目标。

### 7. 目标编辑
- 支持编辑：
  - 今日阅读目标（单位：分钟）
  - 年度阅读目标（单位：本）
- 修改目标后首页卡片需要自动刷新，不允许出现二次手动刷新步骤。

## 需求边界
- 所有数据读取和写入必须经过 `ReadingDashboardRepository`，页面和 ViewModel 不直接访问数据库。
- 首页子卡片属于页面私有组件，不抽到 `UIComponents`。
- 本轮不包含 Android 端已有的打卡、手动计时、长按菜单等完整操作链路。
- 本轮默认验证口径为编译通过与文档闸门通过，不额外引入单元测试。
