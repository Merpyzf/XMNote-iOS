# 在读首页设计文档

## 设计目标
- 用 iOS 原生布局和排版重做 Android `ReadingFragment` 的阅读概览，而不是一比一翻译旧页面。
- 首页视觉上分为“热力图概览 / 趋势信息 / 动作型卡片 / 列表与摘要”四层，保证扫读顺序清晰。
- 所有统计口径统一下沉到 Repository，视图层只消费聚合结果和格式化文案。

## 实际架构

### 1. 页面壳层
- `xmnote/Views/Reading/ReadingContainerView.swift`
  - 负责在读首页、时间线、统计的切换与保活。
- `xmnote/Views/Reading/ReadingDashboardView.swift`
  - 首页真实内容壳层。
  - 在 `.task` 中创建 `ReadingDashboardViewModel`，避免在 `init` 阶段访问环境仓储。

### 2. 状态层
- `xmnote/ViewModels/Reading/ReadingDashboardViewModel.swift`
  - 管理首页 observation、错误态、目标编辑态、跨天刷新。
  - 目标修改后不手动 patch UI，而是依赖 observation 让首页自动回流最新聚合结果。
- `xmnote/ViewModels/Reading/ReadingDashboardFormatting.swift`
  - 负责：
    - 趋势主值数字/单位拆段
    - 今日阅读中轴时长文案压缩
    - 继续阅读与最近在读进度文案
    - 趋势柱图极差场景下的小值可视化映射

### 3. 数据层
- `xmnote/Domain/Repositories/RepositoryProtocols.swift`
  - 定义 `ReadingDashboardRepositoryProtocol`。
- `xmnote/Data/Repositories/ReadingDashboardRepository.swift`
  - 聚合以下首页数据：
    - 今日阅读目标
    - 三项趋势卡
    - 继续阅读书籍
    - 最近在读列表
    - 年度目标与年度已读列表
  - 写入以下目标真相源：
    - `read_target(type = 1)` 每日阅读目标
    - `read_target(type = 0)` 年度阅读目标

## 关键 UI 决策

### 1. 趋势总卡用单卡三分栏，而不是三张独立卡
- 原因：三项指标本质上属于同一组概览信息，拆成三张卡会让页面过碎。
- 实现：`ReadingTrendMetricsSection` 使用单张卡片 + 三栏平分 + 竖向分割线。
- 结果：信息更像一个整体仪表盘，符合首页概览语义。

### 2. 条形图保留轨道，并对小值做非线性抬升
- Android 端条形图存在明显的背景轨道语义，iOS 保留同样的“零值占位”信息。
- 当多根 bar 数据量级差距过大时，直接按线性比例会让小值不可见。
- `ReadingDashboardFormatting.displayedBarRatio` 在短区间内加入 gamma 映射和最小可视高度，保证：
  - 0 值仍然为 0
  - 极小非零值能被看到
  - 大值仍维持相对优势

### 3. 今日阅读使用带底部缺口的 270 度弧环
- 圆弧基于正方形画布绘制，避免横向压缩导致几何变形。
- 文本中轴单独控制宽度，防止品牌数字撑爆弧环内部。
- 目标时长放在底部缺口处，形成“中心读当前，底部读目标”的阅读顺序。

### 4. 继续阅读与今日阅读保持同级卡片比例
- 两张卡共用 `ReadingFeatureCardsStyle` 的比例和边距语义，避免首页中段出现两套节奏。
- 继续阅读为空时不隐藏卡片，而是改为空态引导，保证页面结构稳定。

### 5. 首页错误反馈采用内联横幅，而不是整页打断
- 热力图失败：在热力图卡底部内联提示并提供重试。
- 首页仪表盘失败：在热力图下方插入 `ReadingDashboardInlineBanner`，允许用户关闭。
- 原因：首页是持续浏览页面，错误提示应可感知但不应阻断滚动与已有内容。

## 尺寸与间距策略
- 首页滚动区统一使用设计系统 token：
  - 横向 `Spacing.screenEdge`
  - 顶部 `Spacing.half`
  - 模块间距 `Spacing.base`
  - 大块底部留白 `Spacing.section`
- 趋势总卡内部的纵向分隔留白、图表高度、字体尺寸仍使用布局计算，但卡片之间不再使用“拍脑袋比例值”。
- 今日阅读与继续阅读的卡片比例统一由 `ReadingFeatureCardsStyle.cardAspectRatio` 控制，避免单独魔法数字蔓延。

## 当前已知差距
- iOS 首页暂未补齐 Android 端的打卡、手动计时、长按菜单等强操作链路。
- 年度摘要当前聚焦“目标与已读列表”，未继续承接 Android 端更多二级操作。
- 这些差距属于后续功能补齐，不影响本轮首页仪表盘的核心浏览链路。
