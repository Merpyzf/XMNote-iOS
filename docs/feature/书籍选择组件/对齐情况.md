# 书籍选择组件 iOS ↔ Android 对齐情况

评估日期：2026-04-23

## 功能点清单

1. 统一选书组件配置模型
- 功能点：用统一配置对象描述来源范围、选择模式、默认上下文和创建能力，而不是让每个页面各自拼装选书逻辑。
- iOS 行为：`BookPickerConfiguration` 已统一 `scope`、`selectionMode`、`allowsCreationFlow`、`creationAction`、`defaultQuery`、`preselectedBooks`、`onlineSources`、`preferredOnlineSource` 等维度；`BookSelectionTestViewModel` 已用 20 个场景映射 Android 入口配置。
- Android 行为：`BookSearchSheetFragment.Builder` 已统一 `bookSearchSource(...)`、`supportManualCreate(...)`、`supportMultipleSelect()`、`checkedBookId(...)`、`selectedBookIds(...)`、`defaultSearchKeyword(...)`、`forMatchChapter()`、`what(...)` 等维度。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/Domain/Models/BookPickerModels.swift:98`、`xmnote/ViewModels/Debug/BookSelectionTestViewModel.swift:156`、`xmnote/ViewModels/Debug/BookSelectionTestViewModel.swift:386`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/book/bottom_sheet/BookSearchSheetFragment.kt:343`

2. 书摘编辑页的本地单选 + 创建回填
- 功能点：书摘编辑页可在本地单选一本书，搜不到时进入新增，再把结果回填到编辑页。
- iOS 行为：`NoteEditorView` 以 `scope: .local`、`selectionMode: .single`、`allowsCreationFlow: true`、`creationAction: .nestedSearchPage` 打开 `BookPickerView`；回流时只消费 `.local(BookPickerBook)` 并调用 `selectBook(...)`。
- Android 行为：`NoteEditActivity` 打开 `BookSearchSheetFragment.Builder()`，配置 `bookSearchSource(LOCAL)`、`checkedBookId(...)`、`defaultSearchKeyword("")`、`supportManualCreate(...)`，完成后回填书摘编辑页。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/Views/Note/NoteEditorView.swift:379`、`xmnote/Views/Note/NoteEditorView.swift:391`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/note/activity/NoteEditActivity.kt:517`

3. 本地 / 在线 / 双源范围切换与在线源切换
- 功能点：组件需要支持只看本地、只看在线、以及本地/在线双源切换；在线状态下还能切具体搜索源。
- iOS 行为：`BookPickerConfiguration.scope` 支持 `local / online / both`；`BookPickerViewModel.switchVisibleScope(...)` 保留关键词切换结果层；`selectOnlineSource(...)` 支持在线源切换后自动重搜。
- Android 行为：`BookSearchSheetFragment.Builder.bookSearchSource(...)` 支持 `LOCAL / ONLINE / BOTH`；在线情况下展示 `BookSearchSourceSelectorFragment`，切换当前搜索平台后重新搜索。
- 对齐状态：已对齐。
- 双端代码证据：
  - iOS：`xmnote/Domain/Models/BookPickerModels.swift:98`、`xmnote/ViewModels/Book/BookPickerViewModel.swift:145`、`xmnote/ViewModels/Book/BookPickerViewModel.swift:168`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/book/bottom_sheet/BookSearchSheetFragment.kt:287`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/book/bottom_sheet/BookSearchSheetFragment.kt:324`

4. 在线结果点击后的回流语义
- 功能点：在线搜索结果被点击后，调用方需要得到一个可继续业务流程的结果。
- iOS 行为：`BookPickerOnlineSelectionPolicy.returnRemoteSelection` 支持把在线结果补齐成 `BookPickerRemoteSelection` 后直接回流；`BookPickerSelection` 允许 `.remote` 和 `.local` 共存。
- Android 行为：单选在线结果时，`BookSearchFragment` 会先把 `DoubanBookSearchResult` 拉成 `Book`，然后通过 `sendSearchBookSingleSelectMessage(...)` 回流本地书模型；多选确认时也会先把豆瓣结果拉成 `Book` 后再发送集合。
- 对齐状态：有意差异。
- 双端代码证据：
  - iOS：`xmnote/Domain/Models/BookPickerModels.swift:63`、`xmnote/Domain/Models/BookPickerModels.swift:75`、`xmnote/ViewModels/Book/BookPickerViewModel.swift:280`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/book/fragment/BookSearchFragment.kt:107`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/book/fragment/BookSearchFragment.kt:133`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/book/bottom_sheet/BookSearchSheetFragment.kt:130`
- 评估结论：功能优化。
- Android 反向建议：
  - 是否吸收：可吸收。
  - 建议优先级：中。
  - 风险：中。若 Android 后续也开放“远端结果直返”，需要同步梳理调用方是否能直接消费远端 payload，而不是默认依赖本地 `Book` 模型。

5. 多选过滤场景的空集合确认
- 功能点：过滤型场景能否用空集合表达“全部书籍 / 未限制范围”。
- iOS 行为：`BookPickerMultipleConfirmationPolicy.allowsEmptyResult` 允许 0 选中时仍显示确认栏，并把空集合回流给调用方；`BookSelectionTestViewModel` 已为小组件、复习、批量导出等场景登记这类语义。
- Android 行为：`BookSearchSheetFragment` 多选确认按钮在 `selectedBooks.isEmpty()` 时会直接 toast 并阻止确认，当前没有显式“空集合可提交”能力。
- 对齐状态：有意差异。
- 双端代码证据：
  - iOS：`xmnote/Domain/Models/BookPickerModels.swift:69`、`xmnote/ViewModels/Book/BookPickerViewModel.swift:119`、`xmnote/Views/Book/Sheets/BookPickerView.swift:462`、`xmnote/ViewModels/Debug/BookSelectionTestViewModel.swift:232`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/book/bottom_sheet/BookSearchSheetFragment.kt:130`
- 评估结论：功能优化。
- Android 反向建议：
  - 是否吸收：可吸收。
  - 建议优先级：中高。
  - 风险：低。若 Android 需要补齐该能力，只需新增显式策略位，避免影响现有“必须选中”场景。

6. Android 多入口正式接入覆盖率
- 功能点：Android 多个正式业务入口都已接入统一选书组件，iOS 是否完成同等范围的正式接线。
- iOS 行为：当前正式业务页仅 `NoteEditorView` 已接入统一 `BookPickerView`；另外 19 个 Android 场景目前通过 `BookSelectionTestViewModel` / `BookSelectionTestView` 做配置与结果验证，还没有全部迁到正式页面入口。
- Android 行为：`BookSearchSheetFragment.Builder()` 已在 `NoteEditActivity`、`RelevantListFragment`、`ChapterManagerActivity`、`ReadTimeRecordActivity`、`ReadPlanEditActivity` 等多个正式页面使用。
- 对齐状态：未对齐。
- 双端代码证据：
  - iOS：`xmnote/Views/Note/NoteEditorView.swift:379`、`xmnote/Views/Debug/BookSelectionTestView.swift:11`、`xmnote/Views/Debug/BookSelectionTestView.swift:44`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/note/activity/NoteEditActivity.kt:517`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/note/fragment/RelevantListFragment.kt:259`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/chapter/activity/ChapterManagerActivity.kt:435`
- 评估结论：设计倒退。
- Android 反向建议：
  - 是否吸收：否。
  - 建议优先级：高。
  - 风险：中。当前 iOS 已完成能力收口但正式入口覆盖仍少于 Android，若长期停留在“只在调试中心验证”，会形成用户可见的迁移缺口。

## 总结

- iOS 这一轮已经完成“统一配置 + 统一结果 + 统一状态机 + 20 场景映射验证”的核心收口，组件能力边界已经达到可复用水平。
- 当前主要差距不在能力本身，而在正式业务入口覆盖率；下一阶段应优先把 `RelevantList`、`ChapterManager`、`ReadTimeRecord` 等 Android 已上线入口逐个迁入统一组件。
- `BookPickerSelection.remote` 和“空集合确认”属于 iOS 新增表达力，不应被误写成与 Android 完全同构，但它们是清晰的功能优化，不是语义漂移。
