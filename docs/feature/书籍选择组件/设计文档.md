# 书籍选择组件 - 设计文档

## 设计目标

- 将“书摘编辑中的选书弹层”升级为 Book 模块的统一书籍选择流，而不是继续放大某个页面私有实现。
- 在业务能力上对齐 Android 现有 `BookSearchSheetFragment`，同时使用更符合 Apple HIG 的导航、搜索、状态反馈与任务闭环。
- 把“来源范围切换”“在线搜索源切换”“单选/多选”“手动创建并回填”收敛到同一套组件语义，避免各业务页面各自拼装交互。

---

## 1. 信息架构

书籍选择组件采用“单一任务流 + 分层控制区”的信息架构，而不是把所有筛选入口堆到同一排按钮里。

### 1.1 顶层结构

从上到下分为五层：

1. 导航层
2. 搜索输入层
3. 来源控制层
4. 结果与状态层
5. 多选确认层

### 1.2 各层职责

| 层级 | 职责 | 不负责什么 |
|------|------|------------|
| 导航层 | 提供返回、标题、可选的辅助动作 | 不承载来源切换和搜索源切换 |
| 搜索输入层 | 承接用户关键词输入 | 不负责表达当前来源范围 |
| 来源控制层 | 组织本地/在线范围切换与在线平台切换 | 不直接展示搜索结果 |
| 结果与状态层 | 展示列表、空态、错误态、加载态 | 不负责全局确认动作 |
| 多选确认层 | 在多选模式下显示已选数量与确认动作 | 单选模式下不出现 |

核心原则：控件要按语义分层，避免“一个切换条同时表达本地/在线和具体平台”导致认知混乱。

---

## 2. 关键 UI 决策

### 2.1 将“来源范围”和“在线搜索源”拆成两层控件

这是本组件最关键的交互决策。

| 语义 | 控件形式 | 作用 |
|------|----------|------|
| 来源范围 | segmented control 或等价双态切换 | 决定用户正在看本地结果、在线结果，或允许在两者间切换 |
| 在线搜索源 | 横向 pills / source chips | 决定在线结果具体来自哪个平台 |

原因：

- `local / online / both` 是任务范围语义
- 豆瓣、番茄、起点等是在线平台语义
- 两者若混成一层，会导致用户难以理解“我是在切结果范围，还是在切平台”

因此组件必须使用两层控制：

- 第一层先决定“看本地还是看在线”
- 第二层只在在线结果可见时展示，决定“在线搜哪个平台”

### 2.2 单选点选即完成，多选才出现底栏确认

单选场景的目标是“尽快完成绑定”，不应额外增加确认步骤。

多选场景则需要稳定的集合确认语义，因此：

- 单选：点结果立即回流
- 多选：点结果只改变选中态，底部显示已选数量与确认按钮

这样既对齐 Android 能力，又更符合 iOS 对轻量任务的效率预期。

### 2.3 手动创建必须是空状态中的一级动作

“找不到书”不是异常，而是书籍选择流中的自然分支。因此手动创建不能藏在更多菜单中。

在以下状态必须提供一级 CTA：

- 空书库
- 搜索无结果
- 在线源失败且用户仍想继续完成任务

### 2.4 创建流程保留在同一导航语境内

设计上采用同一 `NavigationStack` 内推进创建页，再返回选择流或直接回填结果。

原因：

- 用户感知上仍处于“选书”任务中
- 不会产生“跳到另一个功能再回来”的割裂感
- 便于保留关键词、已选状态和来源上下文

### 2.5 在线结果行必须带来源标识

当组件支持多个在线平台时，结果行若不标注来源，用户会丢失对结果可信度和后续创建语义的判断。

因此在线结果行需要有明确来源标签，用于表达：

- 当前结果来自哪个平台
- 该平台是否属于纸书源或网文源
- 后续创建时会继承什么默认信息

## 3. 页面结构与布局规则

### 3.1 导航层

- 标题默认使用 `选择书籍`
- 顶部 leading 返回按钮遵循仓库既有导航规范，不额外叠加 glass/material
- 若存在辅助动作，如“取消”“清空选择”，也应保持导航语义，不挤占搜索层

### 3.2 搜索输入层

- 搜索栏固定位于顶部控制区
- 输入提示文案为 `搜索书名、作者、ISBN`
- 关键词在以下场景中都应保留：
  - 切换本地 / 在线
  - 切换在线搜索源
  - 进入手动创建后返回

### 3.3 来源控制层

来源控制层分两部分：

#### A. 来源范围切换

- 当配置为 `local` 时，不显示切换条
- 当配置为 `online` 时，不显示范围切换，但直接进入在线模式
- 当配置为 `both` 时，显示本地 / 在线切换

#### B. 在线搜索源切换

- 仅在当前结果层处于在线语义时显示
- 使用横向滚动 pills 表达多个平台
- 当前选中源需要有稳定高亮态
- 切换后自动重跑在线搜索

在线源集合默认对齐当前项目已有能力：

- 纸间书摘
- 豆瓣读书
- 起点中文网
- 纵横中文网
- 晋江文学城
- 番茄小说
- 长佩文学

### 3.4 结果与状态层

结果层承担三类内容：

- 书籍结果列表
- 状态卡片
- 创建引导

状态优先级：

1. 加载
2. 错误
3. 空书库 / 无结果
4. 正常结果

避免用户在一个区域同时看到互相冲突的反馈。

### 3.5 多选确认层

- 仅在 `multiple` 且存在选中项时显示
- 底栏文案默认使用 `添加所选书籍`
- 需要显示已选数量
- 底栏出现与隐藏应有稳定过渡，但不抢夺内容主层级

## 4. 交互流程

### 4.1 本地单选默认流

1. 用户进入书籍选择组件。
2. 页面展示本地结果或最近相关结果。
3. 用户输入关键词筛选。
4. 用户点选某本书。
5. 组件立即完成并回填调用页。

适用场景：

- 书摘绑定一本已存在的本地书籍
- 阅读计划绑定单本书

### 4.2 本地无结果到手动创建

1. 用户输入关键词后未找到本地结果，或书库本身为空。
2. 页面展示空状态或无结果状态。
3. 用户点击 `手动创建书籍`。
4. 创建完成后：
   - 单选模式直接完成回填
   - 多选模式回到选择器并纳入已选集合

### 4.3 双源切换流

1. 用户进入支持 `both` 的选择器。
2. 默认展示本地或按业务设定的默认来源范围。
3. 用户切换到在线结果。
4. 组件保留关键词并展示当前在线结果。
5. 用户可继续切换在线搜索源，直到找到目标书。

### 4.4 在线搜索源切换流

1. 用户处于在线结果视图。
2. 看到当前选中的在线搜索源 chips。
3. 点击其它搜索源。
4. 组件保留关键词，刷新结果，并明确反馈新的当前源。
5. 若新源失败，用户可重试、换源或直接手动创建。

### 4.5 多选流

1. 用户进入多选模式。
2. 在一个或多个来源下选择书籍。
3. 底栏持续反映当前已选数量。
4. 用户点击确认后，组件统一回流结果集合。

## 5. 状态设计

### 5.1 状态矩阵

| 状态 | 页面主反馈 | 用户主动作 |
|------|------------|------------|
| 首屏加载 | 骨架或加载态 | 等待 |
| 本地结果 | 本地列表 | 搜索、点选、切换在线 |
| 在线结果 | 在线列表 + 当前源高亮 | 搜索、换源、点选、创建 |
| 在线源切换加载 | 列表刷新反馈 | 等待或继续改词 |
| 空书库 | 空状态卡 + 创建 CTA | 手动创建 |
| 无结果 | 无结果卡 + 创建 CTA | 改词、换源、创建 |
| 在线失败 | 错误卡 + 重试 / 换源 | 重试、换源、创建 |
| 创建中 | 提交中反馈 | 等待 |
| 创建完成回填 | 返回结果或回到列表 | 结束或继续选择 |

### 5.2 反馈原则

- 搜索提交后必须立即反馈
- 在线源切换必须有骨架或加载态，不能静默刷新
- 错误反馈要贴近结果区，不打断导航结构
- 创建中要防重复提交

## 6. 与 Android 的关系

### 7.1 需要严格对齐的部分

- 支持 `local / online / both`
- 支持多选
- 支持手动创建
- 支持在线多源检索
- 支持默认关键词与已选回显

### 7.2 iOS 端做得更好的部分

- 单选点选即完成
- 来源范围与在线源显式分层，认知更清晰
- 手动创建是一级动作，不藏入口
- 创建与回填留在同一导航流内，任务更连续

---

## 7. 组件边界

### 8.1 属于组件内部的能力

- 搜索输入与关键词保持
- 来源范围切换
- 在线搜索源切换
- 空态 / 无结果 / 错误态
- 手动创建流转与结果回填
- 单选 / 多选结果语义

### 8.2 不属于组件内部的能力

- 具体在线抓取实现
- 书籍录入页完整业务规则
- 上层业务如何消费回流结果后的后续动作

---

## 8. 风险与规避

### 9.1 风险：把两层来源语义混成一层

后果：

- 用户不知道自己是在切本地/在线，还是在切具体平台
- 页面会出现大量并列 chips，层级混乱

规避：

- 范围切换与在线搜索源切换严格分层

### 9.2 风险：在线结果失败导致任务中断

后果：

- 用户被迫退出当前任务

规避：

- 失败态提供换源、重试和手动创建三条路径

### 9.3 风险：创建返回后丢失上下文

后果：

- 用户重新搜索、重新选择，感知割裂

规避：

- 保留关键词、来源范围、在线源和已选状态
