# 首页书架管理交互设计文档

## 1. 设计目标

首页书架管理是“浏览、查找、轻管理”的工作台，不是 Android 端所有书架管理能力的平铺复刻。iOS 端需要保留 Android 的业务能力和数据修改语义，但在信息架构、视觉密度和操作分层上做重新设计。

核心目标：

- 默认态优先阅读与查找，让用户一眼看到书，而不是先看到管理工具。
- 编辑态才暴露批量管理，避免首页长期处在“工具栏压迫感”中。
- 复杂能力进入 Sheet、二级页或更多菜单，首页只承载高频入口。
- 参考设计稿中的“顶部维度切换 + 书籍网格 + 聚合卡 + 字母索引”框架，但不照搬大胶囊、重卡片和重复封面堆叠。
- UI 优化只发生在交互与视觉层，排序、置顶、删除、分组等数据修改规则必须继续遵循《执行计划》中的 Android 对齐要求。

## 2. 设计原则

### 2.1 呼吸感

- 页面背景使用 `Color.surfacePage`，让书籍封面成为视觉主角。
- 内容区通过留白、分割线、字号层级组织，不依赖重阴影和多层卡片。
- 顶部维度 rail 与内容列表之间保留稳定间距，避免筛选栏贴住封面。
- 聚合卡内部不再套子卡片；封面拼贴、标题、数量属于同一内容块。

### 2.2 简洁

- 首页常驻操作控制在 3 类：添加、搜索、显示设置。
- 批量动作只在编辑态出现。
- 低频动作统一收进“更多”，不在首页横向铺满。
- 维度切换只表达“怎么看书架”，不混入批量修改动作。

### 2.3 易用

- 普通态点击即进入详情或聚合列表。
- 长按单项打开 iOS context menu，承载少量高频单项动作。
- 编辑态底部栏承载批量动作，位置稳定、可预期。
- 搜索态明确禁用手动排序，避免用户误以为过滤结果可以直接落库排序。

## 3. 视觉框架

### 3.1 页面层级

页面从上到下分为四层：

| 层级 | 内容 | 视觉策略 |
| --- | --- | --- |
| 顶部导航层 | 书籍 / 书单切换、添加、搜索、设置 | 沿用现有 `BookContainerView` 顶部结构 |
| 维度 rail | 默认、状态、标签、来源、评分、作者 | 横向轻量切换，选中态品牌绿 |
| 内容层 | 网格、聚合卡、评分分组、作者索引 | 按维度切换布局，不做同构列表 |
| 操作层 | 编辑态底部栏、Sheet、context menu | 普通态隐藏，编辑态浮现 |

### 3.2 背景与表面

- 根背景：`Color.surfacePage`。
- 聚合卡：`Color.surfaceCard`，只使用轻边框或极轻阴影。
- 内容区不使用大面积玻璃；玻璃仅允许出现在导航或功能层，不能铺到书籍内容主体。
- 卡片圆角按角色使用 `CornerRadius`：
  - 封面：`CornerRadius.inlaySmall`
  - 小徽标：`CornerRadius.inlayMedium`
  - 聚合卡：`CornerRadius.blockLarge`
  - Sheet 容器：`CornerRadius.containerMedium`

### 3.3 字体与文案

- 所有生产文本使用 `AppTypography`。
- 维度 rail 使用 `AppTypography.subheadline` 或同等级 token，选中态加字重，不放大到标题级。
- 书名最多两行，作者、日期、数量为次级信息。
- 数量文案短化：
  - `25本`
  - `41条`
  - `未设置标签`
  - `未评分`
- 不在页面内出现解释性长文案；规则说明只出现在禁用态提示、空态或 Sheet 内。

### 3.4 封面

- 所有封面必须使用 `XMBookCover`。
- 默认网格中封面是主视觉，不给整个书籍 item 加白色大卡底。
- 聚合卡中的封面拼贴需要保持真实封面比例，不做强裁切成正方形。
- 封面阴影只保留 Apple Books 式轻量陈列感，不能让阴影先于封面被看见。

### 3.5 书籍列表集合区架构

首页书架管理范围内，所有“书籍列表集合区”统一采用 UIKit 集合视图承载；SwiftUI 只保留页面壳层、导航、Sheet、顶部/底部操作区和单个 cell 的视觉表达。

架构分层：

- SwiftUI 页面壳层：`BookContainerView`、`BookGridView` 负责页面状态、维度切换、编辑态栏、Sheet 与导航路由。
- UIKit 集合区：默认书架和后续二级书籍列表统一使用 `UIViewRepresentable + UICollectionView + UICollectionViewCompositionalLayout` 承接滚动、Grid/List layout、拖拽排序、边缘自动滚动、TabBar 最小化和 iOS 26 bottom scroll edge effect。
- SwiftUI cell：书籍、分组和列表行继续复用 `BookGridItemView`、`BookshelfGroupGridItemView`、`BookshelfDefaultListRow` 等现有视觉组件，通过 `UIHostingConfiguration` 嵌入 `UICollectionViewCell`。

维护约束：

- 生产路径不得再使用 SwiftUI `ScrollView/LazyVGrid/LazyVStack` 手写首页书架管理内的书籍列表与拖拽排序。
- UIKit collection view 只维护本地可见顺序和拖拽预览，不直接访问 Repository 或数据库。
- ViewModel 只接收最终业务意图，例如打开路由、切换选择、提交最终排序 ID 列表。
- 非默认维度和二级只读列表不启用拖拽排序，但仍沿用同一 collection 架构，差异只体现在 layout、cell 配置和数据源。

## 4. 首页信息架构

### 4.1 顶部主切换

沿用当前 `BookContainerView` 的“书籍 / 书单”主切换。书籍管理模块的交互设计只覆盖“书籍”子页；“书单”保持独立演进。

### 4.2 书架维度 rail

新增书架维度 rail：

| 维度 | 作用 | 默认布局 |
| --- | --- | --- |
| 默认 | 展示真实书架顺序，支持手动排序 | 三列封面网格 |
| 状态 | 按阅读状态浏览 | 状态分区 |
| 标签 | 按书籍标签聚合 | 两列聚合卡 |
| 来源 | 按书籍来源聚合 | 两列聚合卡 |
| 评分 | 按评分聚合 | 纵向评分分组卡 |
| 作者 | 按作者首字母聚合 | 字母分区 + 右侧索引 |

rail 设计规则：

- 高度约 44pt，满足触控区域。
- 视觉上比截图中的大胶囊更轻，避免占用过多首屏高度。
- 选中态使用 `Color.brand`，白色文字。
- 未选中态使用弱表面，不使用高对比边框。
- 横向滚动时首尾保留 `Spacing.screenEdge`。
- 切换维度使用 `.snappy`，内容层使用淡入与轻位移，不做夸张弹跳。

`[优化]` Android 端显示/筛选能力更偏设置化，iOS 将高频浏览维度外露为 rail，减少进入长 Sheet 的次数。

## 5. 各维度 UI

### 5.1 默认

默认维度用于呈现真实书架顺序，是唯一承载手动排序的首页维度。

布局：

- iPhone 竖屏默认三列，自适应小屏可降为两列。
- item 不使用外层卡片，采用“封面 + 书名 + 作者/日期”的轻量结构。
- 封面比例统一走 `XMBookCover.aspectRatio`。
- 置顶只显示轻量 pin 标识，不改变整卡背景。
- 书摘数量使用封面角标，弱化为辅助信息。
- 分组 item 使用聚合封面，标题下显示 `N本`。
- 默认维度的集合区以 `BookshelfDefaultCollectionView` 为基准实现，Grid/List 由同一个 `UICollectionView` 通过 Compositional Layout 切换。
- cell 内容继续承载现有 SwiftUI 视觉，滚动、让位动画、拖拽排序和边缘自动滚动交给 UIKit 系统能力。

交互：

- 点击 Book 进入书籍详情。
- 点击 Group 进入组内书籍列表。
- 长按 Book / Group 进入 context menu。
- 进入编辑态后，点击 item 切换选中状态。
- 仅在 `默认 + 手动排序 + 非搜索态 + 编辑态` 允许拖拽排序。
- 拖拽排序只提交松手后的最终顺序，失败时由 ViewModel/Repository 语义恢复。

`[优化]` Android 当前网格中操作能力较重，iOS 默认态不显示批量工具，先保证浏览清爽。

### 5.2 状态

状态维度用于按阅读状态快速回到一类书。

布局：

- 按 `未读`、`在读`、`读完`、`想读`、`搁置`、`弃读` 分区。
- 每个分区头显示状态名与数量。
- 分区内容默认展示横向封面条，最多展示 6 本代表书。
- 分区右侧提供进入箭头，点击查看完整状态列表。
- 空状态分区不展示，除非用户在筛选设置中显式打开“显示空分区”。

交互：

- 点击分区进入状态列表。
- 长按封面仍可打开单项 context menu。
- 状态维度不支持手动排序。

`[优化]` 设计稿中状态被放在顶部 tab，但实际状态适合做“分区浏览”，iOS 采用横向封面条降低重复网格带来的疲劳。

### 5.3 标签

标签维度用于按标签聚合书籍。

布局：

- 两列聚合卡。
- `未设置标签` 固定排在第一位。
- 每张聚合卡包含：
  - 2x3 封面拼贴
  - 标签名
  - `N本` 数量角标
- 拼贴不足 6 本时保持空位留白，不拉伸封面。
- 标签名最多两行，避免长标签挤压封面。

交互：

- 点击聚合卡进入标签书籍列表。
- 长按聚合卡可进入标签管理相关操作，但不在首页直接批量改标签。
- 聚合卡内封面不独立响应点击，避免同一区域出现多重命中。

`[优化]` 设计稿中的标签卡方向正确，但封面重复较重；iOS 保留聚合心智，降低重复封面占比和卡片厚度。

### 5.4 来源

来源维度用于按书籍来源聚合。

布局：

- 复用标签聚合卡结构。
- `未知来源` 固定排在第一位。
- 来源名显示为主标题，数量显示为 `N本`。
- 来源图标不是必需项；如果没有可靠来源图标，不用占位图标制造噪音。

交互：

- 点击来源聚合卡进入来源书籍列表。
- 低频的“设置来源”进入编辑态更多菜单。

`[优化]` Android 支持批量设置来源，但入口较深且偏工具化；iOS 将浏览来源和批量改来源拆开，减少误操作。

### 5.5 评分

评分维度用于按评分快速浏览。

布局：

- 纵向分组卡。
- 分区顺序：未评分、5.0、4.5、4.0、3.5、3.0、2.5、2.0、1.5、1.0、0.5。
- 分区头包含：
  - 星级或 `未评分`
  - 分值
  - 数量
  - 进入箭头
- 分区内容展示横向封面条。
- 分组卡之间保留更大的垂直间距，避免形成压迫列表。

交互：

- 点击分区进入对应评分列表。
- 评分修改不在评分维度直接完成，仍通过单书详情或编辑态批量动作进入。

`[优化]` 设计稿中的评分分组框架可保留，但卡片需要更轻；iOS 使用分割线和留白表达结构，不用厚重白卡堆满页面。

### 5.6 作者

作者维度用于按作者快速定位书籍。

布局：

- 使用字母分区 + 右侧索引。
- 分区标题显示首字母。
- 作者卡两列展示。
- 作者卡包含：
  - 作者名
  - `N本`
  - 2 至 3 本代表封面
- 右侧索引使用轻量文本，不使用大色块。
- 当前索引命中时显示短暂浮层反馈。

交互：

- 点击作者卡进入作者书籍列表。
- 拖动右侧索引快速跳转分区。
- 作者维度不支持手动排序。

`[优化]` Android 已支持按名称/作者等排序，但 iOS 将作者做成显式浏览维度，提升大书库下的定位效率。

### 5.7 二级书籍列表

二级书籍列表包括组内书籍列表，以及从状态、标签、来源、评分、作者聚合入口进入后的完整书籍列表。

布局：

- 二级列表只展示书籍，不展示默认书架顶层 Group。
- 默认使用单列 list layout；如后续支持网格切换，也必须复用同一 `UICollectionView` 架构。
- 视觉 cell 可继续复用 SwiftUI 行视图，通过 `UIHostingConfiguration` 承载。
- 列表底部必须允许内容进入 TabBar 背后，由系统 bottom scroll edge effect 提供过渡，不手写底部蒙版。

交互：

- 点击书籍进入详情。
- 二级只读列表默认不启用拖拽排序。
- 如果后续增加选择或批量操作，选择态仍由 SwiftUI 页面壳层与 ViewModel 管理，collection view 只负责 item 命中与滚动。

实现约束：

- 后续迁移 `BookshelfBookListView` 时，需要与默认模块保持同一架构方向。
- 聚合维度列表、组内列表、状态列表、作者列表只改变数据源和 layout，不另起 SwiftUI 列表架构。

## 6. 普通态交互

### 6.1 点击

| 对象 | 行为 |
| --- | --- |
| Book | 进入书籍详情 |
| Group | 进入组内书籍列表 |
| 标签/来源/评分/作者聚合卡 | 进入对应聚合列表 |
| 维度 rail item | 切换内容维度 |
| 添加按钮 | 打开添加书籍入口 |
| 搜索按钮 | 进入搜索态 |
| 设置按钮 | 打开显示设置 Sheet |

### 6.2 长按

长按 Book / Group 打开 iOS context menu。

Book 菜单：

- 添加书摘
- 置顶 / 取消置顶
- 编辑书籍
- 阅读详情
- 开始计时
- 删除

Group 菜单：

- 置顶 / 取消置顶
- 重命名或编辑分组
- 移出/管理组内书籍
- 删除分组

规则：

- 删除动作使用 destructive 语义。
- context menu 不放批量动作。
- 菜单项数量优先控制在 6 项以内；超过后进入二级更多。

`[优化]` Android 长按菜单承载较多管理动作，iOS 将单项动作留在 context menu，批量动作交给编辑态底部栏。

## 7. 编辑态交互

### 7.1 进入与退出

进入方式：

- 长按 item 后选择“选择”。
- 顶部 action 进入“选择”模式。

退出方式：

- 顶部“取消”。
- 写操作完成后按动作语义决定是否退出；删除完成默认退出，置顶/移动完成保留编辑态。

编辑态顶部：

- 左侧：取消。
- 中间：已选 `N项`。
- 右侧：全选 / 反选。

### 7.2 选择行为

- 点击 item 切换选中。
- Book / Group 可混选。
- 聚合维度下默认选择聚合卡，不直接选择卡内封面。
- 如果进入具体聚合列表，再按 Book / Group 粒度选择。

### 7.3 底部操作栏

底部栏只放高频批量动作：

| 操作 | 规则 |
| --- | --- |
| 置顶 | 对未置顶项生效，已置顶项忽略 |
| 移动 | 打开移动 Sheet，包含移到最前、移到最后、移入分组 |
| 更多 | 标签、来源、阅读状态、收藏、导出 |
| 删除 | destructive 语义，二次确认 |

底部栏视觉：

- 使用功能层表面，不遮住最后一行内容。
- ScrollView 底部需要额外 inset，保证底部沉浸滚动自然过渡。
- 写操作进行中禁用栏内按钮并显示处理态。

`[优化]` Android 批量动作由外层事件分发到 Fragment，入口较分散；iOS 固定为一个编辑态底部栏，用户更容易形成肌肉记忆。

## 8. 手动排序态

手动排序只在以下条件同时满足时启用：

- 当前维度为 `默认`。
- 当前排序模式为 `手动排序`。
- 当前不在搜索态。
- 当前处于编辑态。

排序规则：

- 普通项只能在普通区排序。
- 置顶项不参与普通排序。
- 普通项不得拖入置顶区。
- 拖拽中显示 lifted 状态与目标占位。
- 松手后一次性提交 Repository。
- 写入失败时回滚到拖拽前顺序，并提示失败原因。

Reduce Motion：

- 保留目标占位、边框和透明度变化。
- 减弱位移动画，不依赖弹跳表达位置变化。

`[优化]` Android 拖拽结束后延迟写库以避开 UI 卡顿；iOS 交互上采用即时 UI 反馈、松手一次性提交、失败回滚，但最终写入字段和时间戳规则仍完全对齐 Android。

## 9. 搜索态

搜索态用于查找，不用于排序。

规则：

- 进入搜索后保留当前维度上下文。
- 搜索结果可以按当前维度展示，但禁用手动排序。
- 如果用户尝试排序，展示轻量提示：“搜索结果不支持排序，清除搜索后可调整书架顺序”。
- 搜索为空时展示“没有找到相关书籍”，并提供清除关键词。

`[优化]` Android 端搜索与排序边界不够显性；iOS 直接在交互层阻断过滤结果排序，降低落库语义误解。

## 10. 显示设置

显示设置不再做成所有能力堆叠的长 Sheet，而分为两层：

常用能力外露：

- 维度 rail：默认、状态、标签、来源、评分、作者。
- 当前排序模式可在设置按钮旁或 Sheet 顶部展示。

Sheet 内能力：

- 网格 / 列表。
- 每行数量或封面密度。
- 标题显示方式。
- 是否显示书摘数量。
- 排序规则与升降序。
- 置顶是否在所有排序中固定。

`[优化]` Android 的 `BookShelfDisplaySettingBottomSheet` 承载项目较多，iOS 将“浏览维度”前置，将“展示细节”留在 Sheet，减少设置页负担。

## 11. 删除与分组确认

### 11.1 删除书籍

- 删除单本书籍使用中心确认弹窗。
- 删除按钮使用 destructive 语义。
- 文案说明会删除书籍及关联内容。

### 11.2 删除分组

删除分组必须先询问组内书籍去向：

- 放到书架开头。
- 放到书架末尾。
- 取消。

确认后再执行删除。

`[对齐]` 交互可比 Android 更清楚，但数据层必须保留 Android “先移出组内书籍，再删除分组”的规则。

### 11.3 移入分组

- 通过 Sheet 选择目标分组。
- 支持新建分组。
- 移入后取消置顶、删除旧关系、插入新关系、目标组内追加排序的写入语义继续对齐 Android。

## 12. 空态、加载与错误

### 12.1 空态

| 场景 | 文案 | 主操作 |
| --- | --- | --- |
| 空书架 | 还没有书籍 | 添加书籍 |
| 空搜索 | 没有找到相关书籍 | 清除搜索 |
| 空标签 | 暂无标签分组 | 去设置标签 |
| 空来源 | 暂无来源分组 | 去设置来源 |
| 空评分 | 还没有评分记录 | 查看全部书籍 |

空态只保留一个主操作，避免提供多个并列入口。

### 12.2 加载

- 读取类加载使用现有 `LoadingGate + LoadingStateView` 或 `LoadPhaseHost`。
- 不新增裸 `ProgressView` 作为主加载态。
- 写操作立即显示处理态，并禁用重复提交入口。

### 12.3 错误

- 读取失败展示错误态和重试。
- 写入失败保留当前编辑上下文。
- 排序写入失败必须回滚 UI 顺序。

## 13. 可访问性与适配

- 所有可点元素命中区域不少于 44pt。
- Dynamic Type 下书名最多两行，作者和日期可隐藏或降级。
- 小屏默认网格允许从三列降为两列。
- 聚合卡在大屏可增加列数，但不改变卡内信息层级。
- 作者索引支持 VoiceOver，索引项读作“跳转到 A 分组”。
- context menu 和底部栏操作需要提供清晰的 accessibility label。

## 14. 动效

- 维度切换：`.snappy`。
- 聚合卡进入详情：使用系统导航转场。
- 编辑态进入/退出：底部栏滑入滑出，item 选择态淡入。
- 排序拖拽：普通模式使用 lifted + 占位 + 轻弹性复位。
- Reduce Motion：降低移动幅度，保留占位和边框反馈。

动效只服务结构变化和状态反馈，不做装饰性弹跳。

## 15. Android 对齐与优化清单

| 类型 | 项目 | iOS 方案 | Android 现状 |
| --- | --- | --- | --- |
| `[优化]` | 首页入口密度 | 首页只常驻添加、搜索、设置，批量动作进编辑态 | 批量管理能力通过外层事件和 Fragment 入口分发 |
| `[优化]` | 长按动作 | 单项 context menu 只保留高频动作 | 长按菜单承载较多动作 |
| `[优化]` | 显示设置 | 浏览维度外露，展示细节进 Sheet | `BookShelfDisplaySettingBottomSheet` 承载较多设置 |
| `[优化]` | 标签/来源浏览 | 两列聚合卡，点击进入列表 | 更偏列表与设置组合 |
| `[优化]` | 评分浏览 | 纵向评分分组卡，突出封面预览 | 主要依赖排序/分区能力 |
| `[优化]` | 作者定位 | 字母分区 + 右侧索引 | 主要依赖名称/作者相关排序 |
| `[优化]` | 搜索排序边界 | 搜索态明确禁用排序 | 搜索与排序边界不够显性 |
| `[对齐]` | 手动排序 | 只优化拖拽反馈和失败回滚 | 数据写入继续对齐 Android `book_order` / `group_order` |
| `[对齐]` | 置顶 | 只优化入口与反馈 | 数据写入继续对齐 Android `pinned` / `pin_order` |
| `[对齐]` | 删除分组 | 交互上明确选择组内书籍去向 | 数据写入继续对齐 Android `moveOut + deleteGroup` |

## 16. 后续实现检查点

- 是否所有封面都使用 `XMBookCover`。
- 是否所有生产文本都使用 `AppTypography`。
- 是否没有把内容主体做成大面积玻璃。
- 是否没有在首页堆叠过多常驻操作。
- 是否每个维度都有专属布局，而不是简单换筛选条件。
- 是否在搜索态阻断手动排序。
- 是否在编辑态提供即时写入反馈。
- 是否把所有数据写入规则回到《执行计划》的 `[D]` / `[V]` 核对。
