# 首页书架管理迁移执行计划

## 文档定位

本文用于跟踪 Android 首页书架管理迁移到 iOS 的完整执行过程。迁移允许在产品体验、入口组织、视觉表达和操作反馈上做 iOS 原生优化，但所有会修改数据的操作必须严格对齐 Android 当前实现，避免 iOS 写入后在 Android 端同步、读取或继续编辑时出现语义偏差。

## 数据语义红线

- Android DAO SQL 是数据修改语义源；iOS 不以产品优化为理由改变最终落库结果。
- iOS ViewModel 禁止直接访问数据库；所有读写必须经过 Repository。
- 任何写操作实现前，必须先完成以下核对：
  - Android owner
  - Android DAO SQL
  - 事务边界
  - `is_deleted` 条件
  - `updated_date` 是否更新
  - `last_sync_date` 是否保持不动
  - 影响表范围
  - iOS 对应 Repository 写入点
  - iOS 写入后同步到 Android 的风险结论
- Android 未更新 `updated_date` 的操作，iOS 默认也不更新，避免制造 Android 不会产生的同步事件。
- Android 会更新 `updated_date` 的操作，iOS 必须同步更新，避免 iOS 写入无法被 Android 增量同步识别。
- 如需偏离 Android 数据规则，必须先提交双端代码证据、行为差异、风险评估和回滚方案，获得明确确认后才能落地。

## 产品迁移原则

- 首页不机械复制 Android 的入口密度，优先用 iOS 原生编辑态、上下文菜单、底部操作区和分层设置组织能力。
- Android 首页书架的业务能力需要覆盖，但可以拆分为首页高频动作和二级管理动作。
- 手动排序在 iOS 端使用 `UICollectionView` 系统重排能力承接，不使用 SwiftUI 自建网格拖拽作为生产方向，也不把 `List.onMove` 当作当前三列书架的平替。
- 首页书架管理内所有书籍列表集合区统一对齐默认模块架构：SwiftUI 页面壳层 + UIKit collection view + SwiftUI cell。
- 搜索过滤态默认禁用手动排序，并说明排序需要回到完整书架列表执行。
- 置顶、排序、删除、分组等最终数据结果必须与 Android 完全一致。

## Repository 契约状态

本节最初为实现前草案。2026-05-14 静态审查确认 `BookRepositoryProtocol` 已落地为生产契约，当前应以 `/Users/wangke/Workspace/Xcode/XMNote/xmnote/Domain/Repositories/RepositoryProtocols.swift:10` 为准。核心入口如下：

```swift
protocol BookRepositoryProtocol {
    func observeBookshelf(setting: BookshelfDisplaySetting, searchKeyword: String?) -> AsyncThrowingStream<[BookshelfItem], Error>
    func observeBookshelfSnapshot(settingsByDimension: [BookshelfDimension: BookshelfDisplaySetting], searchKeyword: String?) -> AsyncThrowingStream<BookshelfSnapshot, Error>
    func updateBookshelfOrder(_ orderedItems: [BookshelfOrderItem]) async throws
    func pinBookshelfItems(_ items: [BookshelfItemID]) async throws
    func unpinBookshelfItem(_ item: BookshelfItemID) async throws
    func moveBookshelfItemsToStart(_ items: [BookshelfItemID]) async throws
    func moveBookshelfItemsToEnd(_ items: [BookshelfItemID]) async throws
    func deleteBookshelfItems(_ items: [BookshelfItemID], groupBooksPlacement: GroupBooksPlacement) async throws
    func moveBooks(_ bookIDs: [Int64], toGroup targetGroupID: Int64) async throws
    func batchSetBooksTags(bookIDs: [Int64], tagIDs: [Int64]) async throws
    func batchSetBooksSource(bookIDs: [Int64], sourceID: Int64) async throws
    func batchSetBookReadStatus(bookIDs: [Int64], input: BookshelfBatchReadStatusInput) async throws
    func renameAuthor(oldName: String, newName: String) async throws
    func deleteAuthor(name: String) async throws
    func renamePress(oldName: String, newName: String) async throws
    func deletePress(name: String) async throws
}
```

约束：
- `BookshelfItemID` 必须区分 Book 与 Group，避免误把 Group ID 当 Book ID 写入。
- Repository 接口只接收业务意图和最终顺序，不暴露 GRDB。
- 所有接口实现与开放前必须把对应 Android 数据语义核对任务标为 `[D]`；发现偏离时标 `[!]` 并进入修复或审批。
