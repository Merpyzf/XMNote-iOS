# 首页顶部切换控件统一 - 需求文档

## 1. 背景
- 当前首页顶部存在两套视觉：`ReadingContainerView` 使用引号风格切换，`BookContainerView`/`NoteContainerView` 使用下划线 Tab，`PersonalView`/`SearchView` 为普通标题。
- 视觉不统一，复用性不足，后续维护会重复改样式。

## 2. 目标
- 将 `ReadingTopSwitcher` 抽象为与业务无关的通用组件。
- 首页 5 个页面统一使用同一套顶部视觉语言：
  - `ReadingContainerView`
  - `BookContainerView`
  - `NoteContainerView`
  - `PersonalView`
  - `SearchView`

## 3. 范围
- 范围内：
  - 抽象通用 TopSwitcher 组件。
  - 二级 Tab 页面统一为引号风格切换。
  - 无二级 Tab 页面统一为同风格单标题模式。
- 范围外：
  - 不改页面业务逻辑、数据流、路由。
  - 不调整页面内容区布局与功能行为。

## 4. 需求细则
- `Book`/`Note` 切换控件视觉需与 `Reading` 一致（字重、颜色、引号装饰、动画节奏）。
- `Personal`/`Search` 采用单标题同风格，不强行添加伪 Tab。
- 右侧功能按钮保留原行为与点击区域（44pt 可点区域）。
- 切换动效使用 SwiftUI 原生动画，避免跳变。

## 5. 验收标准
- 五个页面顶部视觉一致，且风格来源统一组件。
- `Reading`/`Book`/`Note` 的 tab 切换可用且逻辑不变。
- `Note` 页右上角图标仍按当前子页在排序/设置之间切换。
- `Personal`/`Search` 顶部标题改造后行为无回归。
