# 首页顶部切换控件统一 iOS ↔ Android 对齐情况

评估日期：2026-03-01

## 功能点清单

1. 顶部切换控件抽象层
- 功能点：是否存在单一可复用顶部切换组件承接多页面。
- iOS 行为：统一抽象 `TopSwitcher`，支持 `tabs/title` 双模式。
- Android 行为：存在 `PillTabs` 组件，但首页主链路仍以页面内各自实现为主，未统一落地到全部容器。
- 对齐状态：未对齐。
- 双端代码证据：
  - iOS：`xmnote/UIComponents/Tabs/TopSwitcher.swift:11`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/compose/components/PillTabs.kt:36`
- 评估结论：功能优化（iOS 组件化程度更高）。
- Android 反向建议：
  - 是否吸收：建议吸收。
  - 建议优先级：中高。
  - 风险：中（需要梳理多页面旧 Toolbar/TabLayout 依赖）。

2. Book/Note/Reading 二级切换统一风格
- 功能点：多子页容器的顶部切换视觉与交互是否统一。
- iOS 行为：`Book/Note/Reading` 容器统一使用 `TopSwitcher`，动画与风格一致。
- Android 行为：典型页面仍以 `TabLayoutMediator + ViewPager` 路径为主，样式与动效分散在各页面。
- 对齐状态：未对齐。
- 双端代码证据：
  - iOS：`xmnote/Views/Reading/ReadingContainerView.swift:60`、`xmnote/Views/Book/BookContainerView.swift:85`、`xmnote/Views/Note/NoteContainerView.swift:85`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/note/activity/NoteManagerActivity.kt:833`
- 评估结论：功能优化。
- Android 反向建议：
  - 是否吸收：建议吸收（先从 Note/Book 主链路试点）。
  - 建议优先级：高。
  - 风险：中（与旧 XML 布局耦合较深，需要阶段迁移）。

3. 单标题页面一致性（Personal）
- 功能点：无二级 Tab 页面是否共享同一顶部设计语言。
- iOS 行为：`PersonalView` 使用 `TopSwitcher(title:)` 与其他页面保持同一语言。
- Android 行为：`PersonalContainerFragment` 仍使用页面内 `TextView` 标题与单独设置按钮组合。
- 对齐状态：未对齐。
- 双端代码证据：
  - iOS：`xmnote/Views/Personal/PersonalView.swift:54`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/main/container/PersonalContainerFragment.kt:245`
- 评估结论：功能优化。
- Android 反向建议：
  - 是否吸收：建议吸收。
  - 建议优先级：中。
  - 风险：低中（主要是视觉层与事件挂接改造）。

4. 搜索页顶部表达策略
- 功能点：搜索页是否纳入统一 TopSwitcher 体系。
- iOS 行为：因 iOS 26 系统搜索入口，搜索页采用系统搜索栏 + 轻量内容区，不强制套用 TopSwitcher。
- Android 行为：`SearchActivity` 继续使用页面内搜索输入栏 + 分栏结果布局。
- 对齐状态：有意差异。
- 双端代码证据：
  - iOS：`xmnote/Views/MainTabView.swift:131`、`xmnote/Views/MainTabView.swift:250`
  - Android：`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/search/activity/SearchActivity.kt:66`、`/Users/wangke/Workspace/AndroidProjects/XMNote/app/src/main/java/com/merpyzf/xmnote/ui/search/activity/SearchActivity.kt:134`
- 评估结论：功能优化（平台原生优先）。
- Android 反向建议：
  - 是否吸收：按平台策略决定，不建议直接照搬。
  - 建议优先级：低。
  - 风险：中（会影响 Android 用户对搜索页独立入口的既有认知）。
