# 阅读日历 - 需求文档

## 1. 背景
- Android 端阅读日历已具备“按日展示阅读行为 + 书籍事件条 + 跨周分段”的核心能力。
- iOS 端已完成基础迁移，本轮聚焦交互与视觉收敛，避免 Android 控件形态直译。
- 本轮目标是完成“月份切换方式重构 + 设置入口 + 三模式切换”的 UI 方案落地。

## 2. 目标
- 顶部月份切换改为“点击月份标题 -> 菜单跳月”，移除左右箭头按钮。
- 顶部控制区收敛为单行：左侧月份选择，右侧模式切换。
- 新增页面级设置入口（导航栏右上角齿轮）与设置页交互骨架。
- 新增模式切换操作：
  - 热力图
  - 活动事件
  - 书籍封面
- 在不改动核心数据语义的前提下，完成可编译、可交互的 iOS 原生界面。

## 3. 范围
### 3.1 范围内
- 月份标题菜单切月。
- 三模式切换控件与网格模式化展示。
- 设置入口按钮与设置弹层 UI。
- 现有加载/空态/错误态保持可用。

### 3.2 范围外
- 设置项持久化与业务生效。
- 新增模式对应的数据查询策略调整。
- 日详情抽屉、筛选器、事件点击跳转。

## 4. 交互需求
### 4.1 月份切换
- 顶部不再提供左右切月按钮。
- 仅允许通过点击月份标题进行菜单选择切月。
- 月份选择区域不展示日历图标，仅保留月份文本与下拉箭头。
- 月份选择区域不使用液态玻璃背景，保持轻量文本触发样式。
- 分页滑动切月继续保留，与菜单选择共用同一状态源。

### 4.2 设置入口
- 导航栏右上角展示设置按钮（`gearshape`）。
- 点击后以 sheet 打开“阅读日历设置”。
- 设置页当前为 UI 设计稿，可包含模式选择与视觉偏好开关占位，不要求接入业务。

### 4.3 模式切换
- 在日历面板顶部右侧增加 segmented 控件（图标分段）。
- 提供 3 个模式：`热力图 / 活动事件 / 书籍封面`。
- 模式切换应有动画反馈，并即时作用于网格显示。

## 5. 展示规则
- 热力图模式：按书籍数量与读完状态显示强度块。
- 活动事件模式：保留跨周事件条、lane、`+N` 溢出策略。
- 书籍封面模式：每日显示最多 3 个封面位，占位方案可先使用视觉色块，超出显示 `+N`。
- 今日/选中/未来日期语义保持一致。

## 6. 验收标准
- 阅读日历顶部不再出现左右切月按钮。
- 可通过点击月份标题切换任意可用月份。
- 顶部控制区保持单行：左月份、右模式切换，不新增垂直占用。
- 可通过右上角设置按钮进入设置页并返回。
- 三模式切换可见、可切换，且网格内容随模式变化。
- 页面编译通过，基础路径（加载/空态/错误态）不回归。

## 7. 风险与约束
- 当前设置页为交互稿，不代表业务配置已生效。
- 书籍封面模式暂用占位表达，后续接入真实封面数据时需补齐资源策略。
- 多模式共存会增加状态维度，需保持 `displayMode` 单一状态源，避免 UI 与数据不同步。
