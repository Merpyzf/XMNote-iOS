# 阅读日历 - 设计文档

## 1. 设计目标
- 交互重心从“步进切月按钮”切换为“月份菜单直达”。
- 在页面层新增设置入口，和日历内容层解耦。
- 引入三模式显示框架（热力图/活动事件/书籍封面），以同一月数据驱动不同视觉表达。
- 保持 Android 业务语义对齐（自然日聚合、跨周分段、lane 稳定），但采用 iOS 原生交互表达。

## 2. 代码结构
### 2.1 页面层
- `xmnote/Views/Reading/ReadCalendar/ReadCalendarView.swift`
- 职责：
  - 注入仓储、驱动 `ReadCalendarViewModel`。
  - 持有 UI 状态 `displayMode`。
  - 提供导航栏设置按钮并弹出 `ReadCalendarSettingsSheet`。

### 2.2 业务内容层
- `xmnote/Views/Reading/ReadCalendar/ReadCalendarContentView.swift`
  - 日历业务内容壳层：顶部控制区、weekday、分页与状态反馈。
- `xmnote/Views/Reading/ReadCalendar/Components/*.swift`
  - 页面私有子视图：顶部控制区、星期标题、连续阅读提示、内联错误提示。
- `xmnote/Views/Reading/ReadCalendar/Sheets/*.swift`
  - 业务弹层：设置与月总结。

### 2.3 跨模块复用组件层
- `xmnote/UIComponents/Foundation/CalendarMonthStepperBar.swift`
  - 顶部月份菜单触发器：只负责“标题展示 + 菜单跳月”，不含左侧日历图标。
- `xmnote/UIComponents/Foundation/ReadCalendarMonthGrid.swift`
  - 周网格渲染：按 `DisplayMode` 切换内容布局。

### 2.4 ViewModel 层
- `xmnote/Views/Reading/ReadCalendar/ReadCalendarViewModel.swift`
- 保持既有数据与分页逻辑，不引入新的数据访问路径。

## 3. 关键交互设计
### 3.1 月份切换策略
- 以 `pagerSelection` 作为唯一切月状态源。
- 切月入口只有两个：
  - 月份标题菜单选择
  - 分页滑动
- 移除顶部左右切月按钮，减少重复交互入口。

### 3.2 设置入口策略
- 设置按钮放置于 `toolbar(.topBarTrailing)`。
- 点击后以 `sheet` 打开设置页。
- 当前设置页仅承载交互设计，不绑定业务持久化。

### 3.3 显示模式策略
- `ReadCalendarContentView.DisplayMode`：
  - `.heatmap`
  - `.activityEvent`
  - `.bookCover`
- `ReadCalendarMonthGrid.DisplayMode` 与面板模式一一映射。
- 模式切换只改变展示层，不改动月数据请求与缓存。
- 顶部模式入口采用图标分段，确保单行布局下仍可容纳 3 模式。

## 4. 组件 API 设计
### 4.1 CalendarMonthStepperBar
- 输入：`title`、`availableMonths`、`selectedMonth`。
- 输出：`onSelectMonth(Date)`。
- 不再暴露 `onPrev/onNext`。

### 4.2 ReadCalendarContentView
- 新增：
  - `Props.displayMode`
  - `onDisplayModeChanged(DisplayMode)`
- 保留：
  - `onPagerSelectionChanged(Date)`（月份变化唯一回调）
  - `onSelectDate(Date?)`（重复点击支持取消选中）

### 4.3 ReadCalendarMonthGrid
- 新增：`displayMode: DisplayMode`。
- `DayPayload` 新增 `bookCount`，用于热力图强度与封面数量。
- 模式行为：
  - `heatmap`：显示强度块。
  - `activityEvent`：显示事件条 overlay。
  - `bookCover`：显示封面占位色块。

## 5. 视觉与动效
- 顶部控件采用单行布局（左月份、右图标分段），减少纵向占用。
- 月份触发器采用无背景文本样式（去液态玻璃），提升沉浸感并避免控件抢焦。
- 模式切换使用图标 `.segmented`，状态变更走 `.snappy`。
- 月份文案切换使用 `.contentTransition(.numericText())`。
- 月网格结构不变，仅替换 cell 内容区。
- 设置页采用 `NavigationStack + Form`，交互路径符合 iOS 设置页习惯。

## 6. 数据与状态约束
- `displayMode` 为页面单一真值源，面板与设置页共享绑定。
- `pagerSelection` 为月份单一真值源，菜单和分页统一写回。
- ViewModel 继续通过 Repository 访问数据，View 层不直接触库。

## 7. 当前实现边界
- 设置页中的“视觉偏好”开关为占位状态，未接入业务逻辑。
- 书籍封面模式当前使用占位视觉表达，后续可替换为真实封面缩略图。
- 事件色彩仍遵循既有取色流程，仅在 `activityEvent` 模式可见。
