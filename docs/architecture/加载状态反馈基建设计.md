# 加载状态反馈基建设计

## 1. 背景与目标

当前项目包含大量本地查询场景，数据返回速度通常很快。若统一在请求开始瞬间展示 `ProgressView`，会产生明显闪烁，降低视觉观感并干扰阅读流。

本基建目标：
- 保证异步操作“可感知反馈”，杜绝点击无响应。
- 通过读写分级策略，兼顾响应感与稳定观感。
- 提供跨页面复用的最小抽象，避免每个页面重复手写 loading 门闩逻辑。

非目标：
- 不做全局阻塞式 loading 管理器。
- 不替代业务数据状态；数据真相仍在 ViewModel/Repository。
- 不引入跨层依赖或改写现有数据访问边界。

## 2. 核心规则（强制）

### 2.1 读加载（Read）
- 策略：延迟显示 + 最短驻留。
- 默认阈值：`delay = 150ms`、`minimumVisible = 200ms`。
- 目的：抑制毫秒级请求的瞬时闪烁，避免“闪一下”的噪音反馈。

### 2.2 写加载（Write）
- 策略：即时反馈，不做延迟显示。
- 典型场景：保存、删除、提交、恢复等会改变数据的操作。
- 同步约束：写操作期间必须禁用重复触发入口，防止并发重复提交。

### 2.3 组件边界
- 生产页面读取加载统一使用 `LoadingGate + LoadingStateView` 或 `LoadPhaseHost`。
- 禁止新增裸 `ProgressView` 作为读取加载主态。
- 写操作局部即时反馈允许使用轻量 `ProgressView` 或 `LoadingStateView(style: .inline)`。

## 3. 基建构成与职责

- `LoadingIntent`：表达加载意图（`none / read / write`）。
- `LoadingPolicy`：表达显隐策略（延迟、最短驻留、显隐动画）。
- `LoadingGate`：门闩状态机，负责根据意图与策略输出 `isVisible`。
- `LoadPhase`：统一阶段枚举（`placeholder / loading / content / empty / error`）。
- `LoadPhaseHost`：阶段容器，避免页面重复 switch 阶段分支。
- `LoadingStateView`：统一 loading 视觉组件（`inline / card`）。

## 4. 标准接入流程

1. 在页面壳层声明 `@State private var loadingGate = LoadingGate()`。
2. 根据业务状态映射意图：
- 读取中 -> `.read`
- 写入中 -> `.write`
- 空闲 -> `.none`
3. 在 `onAppear` 与关键 `onChange` 中同步 `loadingGate.update(intent:)`。
4. 在 `onDisappear` 中执行 `loadingGate.hideImmediately()`，避免离场残留。
5. 内容区渲染策略：
- 页面级主态：`LoadPhaseHost` + `LoadingStateView`。
- 局部写操作：行内或按钮区域展示 `LoadingStateView(style: .inline)`。

## 5. 场景策略矩阵

| 场景 | 意图 | 展示策略 | 交互策略 |
|---|---|---|---|
| 首屏读取 | `read` | 150ms 延迟 + 200ms 最短驻留 | 页面保持可用，避免无意义阻塞 |
| 下拉刷新/分页读取 | `read` | 局部 loading，避免全屏遮挡 | 保留当前上下文滚动与操作 |
| 表单保存 | `write` | 立即显示加载态 | 禁用提交按钮与关键输入 |
| 删除/恢复 | `write` | 立即显示加载态 | 禁用重复点击，防止重复请求 |
| 本地极速查询 | `read` | 优先无感，必要时轻量 inline | 不强制全屏 loading |

## 6. 常见误区与防线

- 误区：所有异步都立即展示 spinner。
  防线：读取类请求必须走读加载策略，避免闪烁噪音。
- 误区：写操作也走延迟显示。
  防线：写操作必须即时反馈，优先防止误触与重复提交。
- 误区：页面离场不回收 gate。
  防线：统一在 `onDisappear` 调用 `hideImmediately()`。
- 误区：阶段与门闩并存时互相覆盖。
  防线：先定义 owner（阶段归业务状态，门闩归可视反馈），按单向映射接入。

## 7. 验收清单

- 点击触发异步后，用户在 0.2s 内可感知反馈或明确禁用态。
- 本地快速读取场景不出现“闪一下”的 loading 噪音。
- 写操作不会重复触发，提交路径无竞态重复请求。
- 页面 push/pop 与切页返回时，不出现 loading 残留。
- 编译通过并且无新增 UI 行为回退。
