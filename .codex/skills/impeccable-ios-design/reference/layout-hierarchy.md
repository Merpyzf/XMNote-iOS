# Layout And Hierarchy

## 目标

让用户一眼知道哪里最重要、什么属于同一组、什么可以操作。

## 规则

- 优先用留白、对齐、字重、尺寸和分割线做层级，不要先加更多容器。
- 卡片内部避免再套子卡片；如果确实是同一张总卡中的多个区块，使用分栏和 divider，而不是再做三张小卡。
- 亲密性优先于平均分布。相关文字、数字、图形应靠近；不相关信息需要明确拉开。
- 不要让页面充满等高、等宽、等结构的指标块；这会快速产生模板感。
- 组件必须适配不同容器宽度、动态字体、iPhone 横竖屏和安全区。
- 触控元素保证 44pt 交互热区，即使视觉尺寸更小也要补足点击区域。
- 若布局需要几何计算，优先相对布局与阈值更新，避免几何抖动和过度依赖 `GeometryReader`。

## 反模式

- 卡片套卡片
- 所有区块都使用同样的上下 padding 和同样的标题结构
- 为了整齐牺牲层级，导致主信息和次信息同权
- 依赖屏幕绝对宽度或 `UIScreen.main.bounds`
