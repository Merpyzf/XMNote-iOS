# Motion And Interaction

## 目标

让状态变化更清楚、更顺手，而不是更花。

## 规则

- 动效优先解释结构变化：展开、收起、切换、滚动沉浸、焦点转移。
- SwiftUI 默认优先 `.snappy`、`.smooth`、`.spring`；只有明确需要时才自定义更复杂时序。
- 进入动画通常比退出动画稍长，但都要保持克制，避免拖沓。
- 任何按钮点击、筛选切换、异步提交，都要立即给出可见反馈。
- 支持 `Reduce Motion`；如果动画承担信息表达责任，要提供低运动量替代，而不是直接消失。
- 交互控件使用 `Button`、`Toggle`、`NavigationLink` 等原生语义控件，避免靠 `onTapGesture` 模拟。
- 触控界面不要依赖 hover 才能发现功能。

## 反模式

- 为了显得灵动到处加弹跳
- 用布局抖动代替清晰反馈
- 状态切换没有 loading / disabled / error 态
- 只有视觉稿动效，没有可访问性与性能判断
