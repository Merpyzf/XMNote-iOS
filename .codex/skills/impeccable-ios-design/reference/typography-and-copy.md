# Typography And Copy

## 目标

在 iOS 原生阅读体验里建立清晰层级，而不是追求 Web 海报感。

## 规则

- 正文字体、交互字体优先保持系统字体体验，避免为了“设计感”破坏可读性。
- 品牌字体仅用于高价值焦点位：主标题、关键数字、品牌标签。正文和密集说明默认不要强制品牌字体。
- 中文界面优先检查字重、行高、字面大小和数字风格，不要只看英文字体搭配。
- 数值文本需要和单位拆层时，优先保证读数速度：数字主导，单位退后，但两者仍应视为同一组。
- 次级标题、说明文案、统计口径必须靠近其主值；若需要拉开距离，必须有分割线或层级变化来解释。
- 文案应减少重复：不要用标题再复述组件名称，不要在说明里再重复用户已能从图形看出的事实。

## 反模式

- 为每个标题都配一个大图标
- 所有文字都同样居中、同样字重、同样颜色
- 品牌字体铺满整个页面
- 为了“精致”把辅助文案做得过淡，导致可读性不足
